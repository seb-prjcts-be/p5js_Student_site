Recursie betekent dat een functie zichzelf opnieuw oproept. Dat klinkt vreemd, maar het is een krachtige manier om vertakkingen, fractals en herhaling op meerdere schalen te maken.

Kernpunten
  • Een functie zichzelf laten oproepen met een kleinere waarde
  • Een duidelijke stopvoorwaarde schrijven om eindeloze recursie te voorkomen
  • Tekenen op meerdere schalen met push(), pop(), rotate() en translate()
  • Het verschil voelen tussen een for-lus en een recursieve aanpak
  • Zien waarom fractals, bomen en bliksem vaak recursief voelen

Waarom dit concept belangrijk is

Recursie laat zien dat een rijke vorm niet altijd veel verschillende regels nodig heeft. Soms is een enkele regel genoeg, zolang die zichzelf opnieuw mag toepassen.

Denkvraag: hoe kan iets oneindig rijk lijken terwijl het steeds hetzelfde principe herhaalt?

De stopvoorwaarde

Een recursieve functie moet weten wanneer ze stopt. Zonder stopvoorwaarde blijf je eindeloos doorgaan.

function telAchteruit(n) {
  if (n <= 0) return;
  print(n);
  telAchteruit(n - 1);
}

Hier roept de functie zichzelf op met een kleinere waarde. Dat is de kern van recursie.

Een recursieve boom

Bij een boom teken je eerst een stam. Daarna teken je vanuit het einde twee kleinere takken. Elke tak doet opnieuw hetzelfde:

function tak(lengte) {
  if (lengte < 12) return;

  line(0, 0, 0, -lengte);
  translate(0, -lengte);

  push();
  rotate(0.4);
  tak(lengte * 0.7);
  pop();

  push();
  rotate(-0.4);
  tak(lengte * 0.7);
  pop();
}

Recursie versus lus

Een for-lus is handig als je iets een vast aantal keer wil herhalen. Recursie is sterk wanneer een vorm of idee zichzelf op kleinere schaal moet herhalen. Daarom zie je recursie vaak bij bomen, rivieren, bliksem en fractals.
