Je sketch leeft in de browser, maar vroeg of laat wil je iets delen: een poster, een GIF voor Instagram, een videoclip voor een project. Er zijn drie manieren om dat te doen, elk met een eigen gebruik.

Kernpunten
  • Ingebouwd: saveCanvas() voor PNG/JPG en saveGif() voor korte GIF-animaties
  • Frame-per-frame opnemen met frameCount en nf() voor zero-padding
  • p5.capture: eenvoudige UI-knopjes voor video- en GIF-opname tijdens je sketch
  • p5.export: verliesvrije, frame-perfecte MP4-export - ook als je sketch traag is
  • Welk systeem past bij welk doel: stilstaand beeld, snelle loop, of publicatie-kwaliteit

Welk systeem wanneer?

  • Eén beeld of een paar frames? Gebruik de ingebouwde saveCanvas().
  • Snel een GIF of WebM delen? Gebruik p5.capture met de UI-knopjes.
  • Lossless MP4 voor een publicatie, Instagram Reel of videoproject? Gebruik p5.export.

1. Ingebouwd: saveCanvas, saveGif & frames

p5.js zelf heeft alles aan boord om een canvas op te slaan. Geen extra library nodig.

Statische afbeelding met saveCanvas()

saveCanvas() downloadt het huidige canvas als PNG of JPG:

function setup() {
  createCanvas(600, 600);
  background(240);
  // ... teken je beeld ...
  saveCanvas('mijn-beeld', 'png');
}

Roep saveCanvas() onderaan in setup() voor een statisch beeld, of in een event zoals mousePressed() om op commando op te slaan:

function mousePressed() {
  saveCanvas('snapshot-' + frameCount, 'png');
}

Frame-per-frame opnemen

Voor animatie kun je elke frame apart opslaan en later samenvoegen tot video met bijvoorbeeld ffmpeg of DaVinci Resolve.

Let op

Zet frameRate() laag (bv. 2) voordat je elke frame opneemt. Anders schrijft je browser honderden bestanden per seconde en loopt alles vast.

Met de functie nf() (number format) zet je frameCount om in 0001, 0002, ... zodat je bestanden in de juiste volgorde staan:

function setup() {
  createCanvas(600, 400);
  frameRate(2);          // traag, zodat opname kan bijhouden
}

function draw() {
  background(240);
  // ... je animatie ...

  // 4 cijfers, geen cijfers na de komma: 0001, 0002, ...
  saveCanvas('frame-' + nf(frameCount, 4, 0), 'png');

  if (frameCount >= 120) noLoop();  // stop na 120 frames
}

Resultaat: frame-0001.png, frame-0002.png, ... Later samenvoegen met ffmpeg:

ffmpeg -framerate 30 -i frame-%04d.png -c:v libx264 -pix_fmt yuv420p animatie.mp4

GIF opnemen met saveGif()

Sinds p5.js 2.x is saveGif() ingebouwd. Je geeft een bestandsnaam en het aantal seconden op:

function keyPressed() {
  if (key === 'g') {
    saveGif('mijn-loop', 3);  // 3 seconden opnemen
  }
}

Extra opties zijn mogelijk via een options-object:

saveGif('mijn-loop', 3, {
  units: 'seconds',   // of 'frames'
  delay: 0,           // start-vertraging in seconden
  silent: false       // toon voortgangsbalk
});
Weetje

saveGif() is bedoeld voor korte loops (enkele seconden). Voor langere of hoge-kwaliteit video's zijn p5.capture of p5.export beter.

Voordelen & grenzen

  • Geen externe library nodig - werkt overal waar p5.js werkt.
  • Frame-sequentie geeft maximale kwaliteit (elk frame is een aparte PNG).
  • Maar: geen video-container, geen audio, en voor GIF beperkt tot korte clips.

2. p5.capture: video-opname met UI-knoppen

p5.capture is een externe library van tapioca24 die een klein bedieningspaneeltje aan je sketch toevoegt: een rode opnameknop, een formaat-selector en een voortgangsindicator.

Installatie

Voeg de library toe in je index.html, na p5.js:

<script src="https://cdn.jsdelivr.net/npm/p5@2/lib/p5.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/p5.capture@1.4.1/dist/p5.capture.min.js"></script>

Meer hoef je niet te doen: de knopjes verschijnen automatisch linksboven in je sketch.

Ondersteunde formaten

  • WebM - video voor het web (kleine bestanden, Chrome-vriendelijk)
  • GIF - animatie, ideaal voor social media
  • MP4 - universele video (vereist extra encoder)
  • PNG-sequentie - frames als ZIP-bestand
  • JPG-sequentie - frames als ZIP-bestand
  • WebP - efficiente animatie-container

Programmatisch opnemen

Naast de knoppen kun je ook vanuit code starten en stoppen:

function setup() {
  createCanvas(600, 600);
}

function keyPressed() {
  if (key === 'r') P5Capture.getInstance().start({
    format: 'gif',
    framerate: 30,
    duration: 180,      // aantal frames
    quality: 1.0
  });
  if (key === 's') P5Capture.getInstance().stop();
}

Wanneer gebruiken?

  • Je wilt snel een GIF of WebM exporteren zonder installatie-gedoe.
  • Je hebt een real-time sketch die al op de gewenste framerate draait.
  • Voor een clip voor Instagram Stories, een Discord-kanaal of een snelle demo.

Bij een zware sketch (veel deeltjes, WEBGL) zakt de effectieve framerate en loopt je opname trager dan je bedoelt. Voor publicatie-kwaliteit: gebruik p5.export.

3. p5.export: verliesvrije, frame-perfecte MP4

p5.export is een tool van prjcts.be die een fundamenteel ander probleem oplost dan saveCanvas() of p5.capture: offline encoding. Elk frame wordt eerst volledig gerenderd, dan pas geencodeerd. De wall-clock tijd van je browser is dus losgekoppeld van de playback-tijd van de video.

Concreet: zelfs als je sketch 5 fps haalt in de browser, krijg je een vloeiende 60 fps MP4-video. Er gaan geen frames verloren en er worden geen artefacten toegevoegd door real-time compressie.

Weetje

p5.export is geinspireerd op p5.capture (MIT, tapioca24), maar gebruikt Mediabunny + de WebCodecs API voor H.264-encoding en de File System Access API om direct in je projectmap te schrijven. Bron: github.com/seb-prjcts-be/p5.export.

Wat kan je ermee?

  • Lossless MP4 (H.264) met gegarandeerde framerate - geen dropped frames
  • PNG-sequentie voor maximale kwaliteit (later samenvoegen met ffmpeg)
  • Drie profielen: Social (Instagram), Edit master, Lossless
  • Automatisch opslaan in [projectmap]/export/YYYYMMDD_HHMMSS.mp4
  • Editor-modus: voer een URL van editor.p5js.org in en exporteer zonder lokale installatie

Twee werkmodi

  1. File System Access (aanbevolen)

    Je selecteert je projectmap in Chrome of Edge. p5.export krijgt lees/schrijfrechten en plaatst je exports direct in een export/-submap. Geen uploads, geen downloads - alles lokaal.

  2. Editor-modus

    Werk je online in editor.p5js.org? Plak de publieke URL van je sketch of download de ZIP en sleep die in de drop-zone. p5.export laadt je project in een iframe, injecteert een path-map voor loadImage en loadJSON, en rendert.

Workflow

  1. Open p5.export

    Ga naar prjcts.be/services/p5.export in Chrome of Edge (File System Access API is nodig).

  2. Kies je bron

    Klik op Pick folder... voor een lokale map, of plak een editor.p5js.org-URL.

  3. Stel je profiel in

    Kies Social (3.5 Mbps, Instagram-ready), Edit master (20 Mbps, post-production) of Lossless PNG. Stel FPS (24/30/60) en duur (seconden) in.

  4. Record

    Klik op Record. Elk frame wordt in je eigen tempo geteld - niet in realtime. De export loopt tot het gewenste aantal frames.

  5. Klaar

    Je krijgt YYYYMMDD_HHMMSS.mp4 in je export/-map. Geen verdere actie nodig.

Instagram-tuning

Het Social-profiel is specifiek getuned voor Instagram:

  • moov-atom aan het begin van het bestand (faststart voor streaming)
  • mp42-brand (iOS/Instagram herkent dit beter dan standaard MP4)
  • Keyframe elke seconde geforceerd (betere scrubbing en upload)
  • Video-only, geen stille audio-track (kleinere bestanden)
  • Aanbevolen canvas: 1080x1080 (feed) of 1080x1920 (Reels/Stories)

API voor eigen integratie

Wie p5.export wil aanroepen vanuit eigen code kan de publieke API gebruiken:

const canvas = document.querySelector('canvas');

const { tick, stop } = p5export.startCapture({
  canvas: canvas,
  frames: 300,          // totaal aantal frames
  playbackFps: 60,      // framerate van de output-video
  profile: 'social',    // 'social' | 'edit' | 'master'
  format: 'mp4',        // 'mp4' | 'png'
  bitrateMbps: 20,
  onDone: (blob, info) => console.log('klaar', info),
  onFrame: (n) => console.log('frame', n)
});

// In je draw-loop, na het tekenen:
function draw() {
  // ... je sketch ...
  tick();               // tel dit frame mee
}

// Stoppen kan ook handmatig:
// stop();

Wanneer gebruiken?

  • Je wilt publiceren op Instagram, een festival-submission, of een archief.
  • Je sketch is te zwaar om in realtime op 60 fps te lopen.
  • Je wilt exact controle over duur, FPS en bitrate.
  • Je wilt geen compressie-artefacten van real-time encoders.
Let op

p5.export vereist een Chromium-browser (Chrome, Edge, Brave) vanwege de File System Access API. Firefox en Safari ondersteunen deze API nog niet volledig.
