Sol LeWitt (1928–2007) was een Amerikaanse conceptueel kunstenaar die beroemd werd om zijn Wall Drawings: kunstwerken die bestaan uit geschreven instructies. Hij stuurde die instructies naar musea, die ze dan ter plekke lieten uitvoeren door medewerkers of bezoekers. De tekening zelf was bijzaak — de instructie was het werk.

De instructie als kunstwerk

LeWitt geloofde dat het idee achter een kunstwerk belangrijker is dan de uitvoering ervan. Zijn instructies waren soms één zin, soms meerdere pagina's. Elke uitvoering was anders, maar de instructie bleef dezelfde.

Een bekende instructie:

"Lines from corners, sides, and center of the canvas to random points on the wall."

Dit is niets anders dan een algoritme. Code is een instructie. Wanneer je een p5.js-sketch schrijft, doe je precies wat LeWitt deed: je beschrijft een systeem dat iemand anders — in dit geval de computer — uitvoert.

Instructie → code

Neem deze instructie: "Verbind 40 willekeurig geplaatste punten met rechte lijnen naar hun drie dichtstbijzijnde buren."

Stap voor stap vertaald naar code:

// Stap 1: genereer willekeurige punten
let punten = [];
for (let i = 0; i < 40; i++) {
  punten.push({ x: random(width), y: random(height) });
}

// Stap 2: voor elk punt — zoek de 3 dichtstbijzijnde
for (let a of punten) {
  let gesorteerd = punten
    .filter(b => b !== a)
    .sort((b, c) => dist(a.x, a.y, b.x, b.y) - dist(a.x, a.y, c.x, c.y));

  // Stap 3: teken een lijn naar elk van de 3 dichtste buren
  for (let i = 0; i < 3; i++) {
    line(a.x, a.y, gesorteerd[i].x, gesorteerd[i].y);
  }
}

Raster met willekeurige diagonalen

Veel van LeWitts werken zijn gebaseerd op rasterverdeling met een simpele variabele: de richting van een diagonaal. De instructie: "Verdeel het canvas in een raster. Teken in elke cel een diagonale lijn in een willekeurig gekozen richting."
