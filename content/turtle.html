Turtle geometry is tekenen met een denkbeeldige schildpad. Je geeft alleen eenvoudige commando's zoals vooruit en draaien, maar uit die korte instructies kunnen complexe figuren groeien.

Waarom dit concept sterk is

Turtle geometry maakt code heel concreet. Je schrijft geen abstracte formules, maar stuurt een tekenend lichaam door de ruimte. Daardoor voelt algoritmisch denken bijna lichamelijk aan.

Denkvraag: hoe kan een rijke tekening ontstaan uit maar drie of vier regels?

Wat je hier leert

  • ruimte beschrijven met positie en hoek
  • patronen bouwen met herhaling
  • functies maken die je eigen tekentaal vormen
  • zien hoe een regel niet alleen rekent, maar ook tekent

Oorsprong: Logo en Seymour Papert

Turtle graphics werden bekend via de programmeertaal Logo. Seymour Papert gebruikte het idee om leerlingen wiskunde te laten begrijpen door beweging en actie.

De schildpad heeft drie eigenschappen:

  • Positie - x en y op het canvas
  • Richting - de hoek waarnaar hij kijkt
  • Pen - neer (tekent) of op (beweegt zonder lijn)

De basiscommando's

forward(n)    // Ga n pixels vooruit en teken een lijn
right(hoek)   // Draai n graden naar rechts
left(hoek)    // Draai n graden naar links
penUp()       // Pen omhoog: beweeg zonder te tekenen
penDown()     // Pen omlaag: begin weer te tekenen

Implementatie in p5.js

De turtle houdt zijn positie en richting bij. forward() berekent het nieuwe punt met sinus en cosinus:

let tx, ty, heading, penNeer;

function forward(n) {
  let nx = tx + n * cos(radians(heading));
  let ny = ty + n * sin(radians(heading));
  if (penNeer) line(tx, ty, nx, ny);
  tx = nx;
  ty = ny;
}

function right(hoek) { heading += hoek; }
function left(hoek)  { heading -= hoek; }

Van regel naar patroon

De kracht van turtle geometry zit in herhaling. Een vierkant is niets meer dan vier keer vooruit en 90 graden draaien.

for (let i = 0; i < 4; i++) {
  forward(100);
  right(90);
}

Een ster met vijf punten gebruikt een andere draaihoek:

for (let i = 0; i < 5; i++) {
  forward(120);
  right(144);
}

Voor een regelmatige n-hoek gebruik je altijd 360 / n:

let n = 6;
for (let i = 0; i < n; i++) {
  forward(80);
  right(360 / n);
}

Spiralen en afwijkingen

Vergroot de staplengte elke iteratie een beetje, dan krijg je een spiraal:

for (let i = 0; i < 200; i++) {
  forward(i * 0.5);
  right(91);
}

Dat kleine verschil tussen 90 en 91 graden lijkt onbelangrijk, maar verandert het gedrag volledig. Dat is typisch voor codeconcepten: kleine regels kunnen grote visuele gevolgen hebben.

Recursieve patronen

Als je dezelfde tekening op kleinere schaal laat terugkeren, kom je uit bij recursie en fractals:

function boom(lengte) {
  if (lengte < 5) return;
  forward(lengte);
  right(30);
  boom(lengte * 0.7);
  left(60);
  boom(lengte * 0.7);
  right(30);
  forward(-lengte);
}

boom(80);

Hier zie je dat turtle geometry niet alleen over vormen gaat, maar ook over structuur en groei.

Oefening

Maak drie versies van hetzelfde idee:

  • een nette vorm met een "juiste" hoek, zoals 90 of 120 graden
  • een versie met een kleine afwijking, zoals 89 of 121 graden
  • een persoonlijke variant met kleur, variabele staplengte of pen-up bewegingen

Vergelijk daarna welk patroon het meest voorspelbaar voelt en welk patroon juist verrassend wordt.
