Data leven meestal buiten je sketch: in CSV-bestanden, JSON-bestanden of online via een API. Zodra je die data in p5.js binnenhaalt, opent er een wereld aan datagestuurde kunst: een klok die pulseert op live meteorietinslagen, een stad die verkleurt op basis van temperatuur, een compositie die groeit terwijl Wikipedia wordt bewerkt.

Kernpunten
  • Verschil tussen CSV, JSON en API - en wanneer je welk formaat gebruikt
  • Async laden in setup() met await loadTable() en await loadJSON() (p5.js 2.x)
  • Data uitlezen: rijen, kolommen en genestelde velden
  • Live data ophalen van een API met loadJSON()
  • Data omzetten naar beeld of geluid - data als creatief materiaal

Over The Edges - data als kunst

In 2000 organiseerde het SMAK in Gent de stadstentoonstelling Over The Edges. De Milanese kunstenaar Alberto Garutti (°1948) liet de straatlichten aan de Gentse Vrijdagmarkt oplichten bij elke nieuwe geboorte in Gent. Elke baby was een flikkering. Jaren later flikkert dit werk nog steeds na in de harten van veel Gentenaars.

Dat is wat datagestuurde kunst kan zijn: een onzichtbare stroom getallen - hier de geboortecijfers van een stad - wordt voelbaar gemaakt in een materiaal dat iedereen begrijpt, licht. Die beweging van data naar beleving is waar dit hoofdstuk over gaat.

Weetje

Garutti's werk staat bekend als "poëtische data-art". De mooiste sonificaties en datavisualisaties doen hetzelfde: ze laten je voelen wat een getal alleen niet kan doen.

CSV - eenvoudige tabellen

CSV staat voor Comma-Separated Values. Het is wellicht het eenvoudigste bestandsformaat dat er bestaat om data op te slaan: rijen met waarden, gescheiden door komma's.

naam,leeftijd,stad
Anne,24,Gent
Frank,31,Brussel
Laila,19,Antwerpen

In p5.js 2.x laad je een CSV asynchroon in setup():

let tabel;

async function setup() {
  createCanvas(600, 400);

  // Wacht tot het bestand geladen is
  tabel = await loadTable('data.csv', 'csv', 'header');

  // Toegang tot data
  let aantalRijen = tabel.getRowCount();
  let aantalKolommen = tabel.getColumnCount();

  console.log(aantalRijen + ' rijen, ' + aantalKolommen + ' kolommen');

  // Doorloop alle rijen
  for (let r = 0; r < aantalRijen; r++) {
    let naam = tabel.getString(r, 'naam');
    let leeftijd = tabel.getNum(r, 'leeftijd');
    console.log(naam + ' is ' + leeftijd + ' jaar');
  }
}

Het derde argument 'header' zegt dat de eerste rij kolomkoppen bevat. Gebruik dan namen zoals 'leeftijd'; anders moet je met indexen werken.

JSON - genestelde data

JSON staat voor JavaScript Object Notation. Het is een lichtgewicht formaat dat gemakkelijk te lezen en te schrijven is. Herinner je het ikke-object uit Objecten & klassen? JSON gebruikt exact dezelfde schrijfwijze:

{
  "stad": "Gent",
  "inwoners": 265000,
  "wijken": ["Binnenstad", "Ledeberg", "Gentbrugge"],
  "oprichting": 630
}

JSON kan dieper nestelen dan CSV: arrays binnen objecten, objecten binnen arrays. Dat maakt het ideaal voor rijkere data zoals locaties met coördinaten of spellen met meerdere spelers.

let data;

async function setup() {
  createCanvas(600, 400);

  data = await loadJSON('stad.json');

  // Direct een veld lezen
  console.log(data.stad);          // "Gent"
  console.log(data.wijken.length); // 3

  // Door een array loopen
  for (let wijk of data.wijken) {
    console.log(wijk);
  }
}
Tip

Bouw altijd eerst je data-lezen op met console.log(). Pas als je ziet wat er binnenkomt, begin je met tekenen. Anders teken je in het luchtledige.

API - live data van een server

Een API (Application Programming Interface) is een set afspraken tussen softwaretoepassingen. Voor jouw sketch betekent het meestal: een URL waar je JSON kunt ophalen die nu, op dit moment, wordt samengesteld.

De functie blijft dezelfde - loadJSON() - maar je geeft een URL mee in plaats van een lokaal bestand:

let kleuren;

async function setup() {
  createCanvas(600, 400);

  // Live data van een publieke API
  kleuren = await loadJSON('https://corpora-api.glitch.me/colors');

  noStroke();
  for (let i = 0; i < kleuren.colors.length; i++) {
    fill(kleuren.colors[i].rgb);
    rect(i * 60, 0, 60, height);
  }
}
Let op

Niet elke API laat zomaar requests uit een browser toe. Als je "CORS error" in de console ziet, betekent dat dat de server jouw domein niet toelaat. Zoek dan een publieke API die wel browser-vriendelijk is (zie hieronder).

Bronnen van open data

Hieronder een paar goede startpunten. Elke link is een schatkist aan materiaal voor je volgende sketch:

  • Corpora - honderden grappige JSON-bestanden: dierennamen, kleuren, mythologie, geuren, sterrenbeelden
  • Corpora API - dezelfde data, maar direct via URL op te halen
  • Open Data Stad Gent - mobiliteit, cultuur, lucht, evenementen
  • Open Data België - duizenden datasets van overheden
  • NASA Open Data - ruimte, klimaat, meteorieten
  • NASA Meteorite Landings API - concreet startpunt voor data-sonificatie

Creatieve toepassingen

Nu je weet hoe je externe data laadt, kun je beginnen met het zoeken naar betekenis in getallen. Een paar ideeën om mee te starten:

  • Een dag uit je leven als data: hartslag, stappen, slaap - en die in beeld zetten
  • Luchtkwaliteit uit Open Data Gent koppelen aan kleur of beweging
  • De geboortecijfers van jouw stad vertalen naar flikkeringen, zoals Garutti deed
  • Een klok die bouwt op datum en tijd + live weerdata
  • Een sonificatie van CSV-cijfers uit een spreadsheet die jou persoonlijk raakt

Niet elke dataset wordt goede kunst. Zoek data die jou iets doen - een buurt, een hobby, een zorg - en laat daaruit iets ontstaan.

Oefening

Kies één JSON-bestand uit Corpora (bv. colors.json of flowers.json). Schrijf een sketch die:

  1. Het bestand async laadt met await loadJSON() in setup().
  2. De eerste vijf items uit de lijst op de canvas tekent als kleurvlakken of tekst.
  3. Bij een muisklik een willekeurig ander item uit de lijst toont.

Schrijf drie zinnen over wat je opmerkte: was de data bruikbaar zoals ze binnenkwam, of moest je er eerst mee worstelen?

Conclusie

Externe data verbindt je sketch met de wereld. Of je nu een CSV uit je eigen spreadsheet laadt, een JSON-bestand van een collega krijgt of live data van een API gebruikt: de mogelijkheden zijn bijna eindeloos. Het mooiste werk ontstaat vaak niet door zo veel mogelijk data te gebruiken, maar door die ene betekenisvolle stroom te vinden en er iets aanraakbaars van te maken.
