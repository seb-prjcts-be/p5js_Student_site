Een L-systeem is een regelsysteem dat tekst herschrijft. Door die tekst daarna als tekening te lezen, krijg je groeivormen zoals planten, vertakkingen en fractals.

Kernpunten
  • Denken in regels en herschrijvingen vanuit een axioma
  • Groei beschrijven met een grammatica in plaats van direct tekenen
  • Een string lezen met turtle-commando's: F, +, -, [, ]
  • Vertakkingen maken met push() en pop()
  • Het verband zien tussen tekststructuur en beeldstructuur

Waarom dit concept boeiend is

Bij een L-systeem teken je een plant niet stap voor stap met de hand. Je beschrijft alleen hoe ze groeit. Het beeld ontstaat dus uit een grammatica.

Denkvraag: kan een beeld een soort taal hebben?

Hoe werkt het?

Je begint met een axioma, een startstring, en vervangt elke letter herhaaldelijk via regels.

Axioma:  F
Regel:   F -> F+F-F-F+F

Iteratie 0: F
Iteratie 1: F+F-F-F+F
Iteratie 2: F+F-F-F+F + F+F-F-F+F - ...

Daarna lees je de string met turtle-commando's:

  • F - ga vooruit en teken
  • + - draai rechts
  • - - draai links
  • [ - sla positie en richting op
  • ] - herstel positie en richting

Plantmodel

Met [ en ] kun je vertakkingen maken:

Axioma: X
Regels: X -> F+[[X]-X]-F[-FX]+X
        F -> FF
Hoek:   25
for (let c of sentence) {
  if (c === "F") { line(0, 0, len, 0); translate(len, 0); }
  else if (c === "+") rotate(radians(hoek));
  else if (c === "-") rotate(radians(-hoek));
  else if (c === "[") push();
  else if (c === "]") pop();
}

Zo krijg je iets dat voelt als groei: elke iteratie bouwt verder op de vorige structuur.

Wat verandert een regel?

Een kleine wijziging in een regel of hoek kan een plant sierlijk, compact, chaotisch of bijna architecturaal maken.

  • meer iteraties - meer detail
  • grotere hoek - bredere vertakking
  • andere vervangregel - ander groeigedrag

Oefening

Verander maar een ding tegelijk: eerst de hoek, dan het aantal iteraties, dan de regel voor X. Beschrijf telkens of je resultaat meer op een plant, een netwerk, een ornament of een abstract patroon lijkt.
