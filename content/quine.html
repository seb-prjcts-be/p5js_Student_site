Een quine is een programma dat zijn eigen broncode produceert zonder een bestand te openen. Het is een klein, vreemd en verrassend idee over zelfverwijzing in code.

Waarom dit concept bijzonder is

De meeste code doet iets buiten zichzelf: tekenen, rekenen, reageren of simuleren. Een quine draait dat om. Het programma wordt zelf het onderwerp.

Denkvraag: kan een systeem zichzelf beschrijven zonder van buitenaf hulp te krijgen?

Wat je hier leert

  • het verschil tussen code en data
  • hoe strings een beschrijving van code kunnen bevatten
  • wat zelfverwijzing betekent in informatica
  • waarom een slim idee soms belangrijker is dan veel code

Zelfverwijzing

Het kernprobleem: hoe schrijf je een programma dat zichzelf beschrijft zonder gewoon een bestand te lezen? Je mag geen readFile() of netwerk gebruiken. De code moet zichzelf dus als data mee dragen.

De truc is quoting: je bewaart een deel van de code in een string en gebruikt die string om de volledige broncode opnieuw op te bouwen.

// Pseudocode quine
let s = "let s = X; print('let s = ' + quote(s) + '; ' + s)";
print("let s = " + quote(s) + "; " + s);

Een toegankelijke versie in JavaScript

In JavaScript heeft elke functie een ingebouwde manier om zijn eigen broncode te lezen: functie.toString().

function setup() {
  let code = setup.toString();
  console.log(code);
}

Dit is geen perfecte quine, omdat je alleen de functie leest en niet het volledige programma. Maar didactisch is het wel een sterke opstap.

Zelfkennis op het canvas

In p5.js kun je een functie letterlijk zichzelf laten tekenen:

function setup() {
  createCanvas(600, 400);
  background(20);

  let code = setup.toString();
  let lijnen = code.split("\n");

  fill(140, 230, 120);
  textFont("monospace");
  textSize(11);

  lijnen.forEach(function(lijn, i) {
    text(lijn, 10, 20 + i * 15);
  });
}

Waarom dit meer is dan een grap

Een quine opent een deur naar diepere vragen: wat is een beschrijving, wat is een representatie, en waar zit de grens tussen systeem en meta-systeem? Daarom hoort dit onderwerp thuis tussen creatieve codeconcepten: het is code als filosofie.

Oefening

Gebruik eerst het voorbeeld met toString(). Laat daarna de code niet alleen zichzelf tonen, maar ook iets extra doen: een raster tekenen, een kleurverloop tonen of sleutelwoorden accentueren. Zo wordt de broncode zelf beeldmateriaal.

Echte quines

Een volledige quine in JavaScript ziet er ingewikkelder uit. Je hoeft die niet uit je hoofd te leren. Belangrijker is dat je begrijpt waarom ze werkt:

(function q(){
  let s = "(function q(){\n  let s = " +
          JSON.stringify(s) + ";\n  " + q.toString().split("\n").slice(2,-1).join("\n  ") +
          "\n})();";
  console.log(s);
})();

Quines bestaan in bijna elke programmeertaal. Ze tonen dat code zichzelf niet alleen kan uitvoeren, maar ook kan representeren.
