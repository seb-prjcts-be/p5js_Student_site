Een quine is een programma dat geen invoer nodig heeft en als uitvoer precies zijn eigen broncode produceert. Het is een van de meest fascinerende zelfverwijzende concepten in de informatica.

Zelfverwijzing

Het kernprobleem: hoe schrijf je een programma dat zichzelf beschrijft, zonder gewoon een bestand te lezen? Je mag geen readFile() of netwerk gebruiken — de code moet zichzelf kennen.

De truc is quoting: je slaat de code op als data, en gebruikt die data om de code + data opnieuw te produceren.

// Pseudocode quine
let s = "let s = X; print('let s = ' + quote(s) + '; ' + s)";
print("let s = " + quote(s) + "; " + s);

De variabele s bevat de code buiten zichzelf — en de print-regel combineert beide delen opnieuw tot de volledige broncode.

JavaScript: toString()

In JavaScript heeft elke functie een ingebouwde manier om zijn eigen broncode te lezen: functie.toString().

function setup() {
  let code = setup.toString(); // Lees eigen broncode
  console.log(code);           // Druk jezelf af
}

Dit is geen echte quine (de volledige broncode bevat meer dan alleen setup), maar het toont het principe van zelfkennis.

Zelfkennis op het canvas

In p5.js kun je een functie letterlijk zichzelf laten tekenen:

function setup() {
  createCanvas(600, 400);
  background(20);

  let code = setup.toString(); // Eigen broncode
  let lijnen = code.split("\n");

  fill(140, 230, 120);
  textFont("monospace");
  textSize(11);

  lijnen.forEach(function(lijn, i) {
    text(lijn, 10, 20 + i * 15);
  });
}

Voer de code uit — de setup()-functie tekent zichzelf op het canvas. Pas de tekstkleur of grootte aan en klik Run. Verandert de weergave mee?

Echte quines

Een volledige quine in JavaScript ziet er zo uit (probeer dit in de browser console):

(function q(){
  let s = "(function q(){\n  let s = " +
          JSON.stringify(s) + ";\n  " + q.toString().split("\n").slice(2,-1).join("\n  ") +
          "\n})();";
  console.log(s);
})();

Bekende quines bestaan in bijna elke programmeertaal. Ze zijn een bewijs van Kleene's recursiestelling: elk berekend programma kan een zelfverwijzende versie van zichzelf produceren.
