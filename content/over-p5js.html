p5.js is een JavaScript bibliotheek die creative coding toegankelijk maakt. Het is gebaseerd op Processing, maar werkt volledig in de browser zonder extra software.

Wat is Processing?

Processing is een programmeertaal en ontwikkelomgeving die in 2001 werd gemaakt door Casey Reas en Ben Fry aan het MIT Media Lab. Het doel was om programmeren toegankelijk te maken voor kunstenaars, designers en beginners.

Processing gebruikt Java en heeft zijn eigen editor. Het is krachtig, maar vereist dat je software installeert.

Van Processing naar p5.js

p5.js is de JavaScript-versie van Processing. Het werd ontwikkeld door Lauren McCarthy en een community van ontwikkelaars. Het belangrijkste verschil:

  • Processing: Java-gebaseerd, desktop applicatie
  • p5.js: JavaScript-gebaseerd, werkt in de browser
  • Processing Python: Python-gebaseerd, alternatieve syntax

Met p5.js hoef je niets te installeren - je schrijft code in een editor en opent het in je browser!

Processing is gratis en open source. Iedereen kan bijdragen aan de software. Je kunt Processing zien als een extra laag bovenop een programmeertaal - een bibliotheek van functies die jou toelaten om gemakkelijk creatieve projecten te maken.

In deze cursus focussen we op p5.js.

De p5.js Editor

De officiële p5.js editor (editor.p5js.org) is een online ontwikkelomgeving waar je direct kunt beginnen met coden. Voordelen:

  • Geen installatie nodig
  • Automatisch opslaan
  • Delen van sketches is eenvoudig
  • Werkt op elke computer met een browser

Je kunt ook je eigen HTML-bestand maken en p5.js via een CDN laden (zoals we in deze cursus doen).

De console

Onder het code-venster in de editor zie je de console. Dat is je informatiescherm: je kunt er waarden in printen, en foutmeldingen verschijnen er automatisch.

// Een waarde tonen in de console:
console.log("Hallo!");
console.log(mouseX);  // Toont de muispositie

Je kunt de console ook gebruiken als rekenmachine: typ een berekening in en druk op Enter.

Vriendelijke foutmeldingen

Het Friendly Error System van p5.js geeft begrijpelijke foutmeldingen in eenvoudige taal - inclusief een hint wat er mis is en hoe je het kunt oplossen. Als er een fout in je code zit, lees de console dus aandachtig!

Browser DevTools

Druk op F12 of Shift + Ctrl + I om de ontwikkelaarstools te openen. Drie handige tabbladen:

  • Console: foutmeldingen en console.log() output verschijnen hier. Je kunt er ook JavaScript uitproberen.
  • Elements: bekijk de HTML-structuur van je pagina en het canvas. Handig om te zien hoe p5.js elementen toevoegt.
  • Sources: je kunt hier breakpoints zetten om je code regel per regel te doorlopen als iets niet werkt.

De Reference

De p5.js Reference is je woordenboek voor alles wat p5.js kan. Voor elke functie vind je:

  • Uitleg wat een functie doet
  • Welke parameters je kunt gebruiken
  • Werkende voorbeelden

Tip: Als je niet weet hoe iets werkt, zoek het op in de Reference. Je hoeft niet alles uit het hoofd te leren!

Schetsen bewaren en organiseren

In de online editor worden je schetsen automatisch opgeslagen als je bent ingelogd. Toch loont het de moeite om goede gewoontes aan te leren:

  • Gebruik duidelijke namen - geen "sketch_01" maar iets als "LES_kleur_rainbow"
  • Geen spaties in bestandsnamen - gebruik een underscore _ of koppelteken -
  • Gebruik versienummers als je experimenten bijhoudt: _v01, _v02
  • Organiseer in collecties - de editor ondersteunt mappen per project of thema
  • Maak regelmatig back-ups - download de bestanden lokaal voor langetermijnsprojecten

Canvas opslaan

Met saveCanvas() sla je het huidige canvas op als afbeelding (PNG of JPG):

function keyPressed() {
  if (key === 's') {
    saveCanvas('mijn-sketch', 'png');
  }
}

Voor een animatie kun je elk frame opslaan met een volgnummer:

function draw() {
  background(220);
  circle(frameCount % width, height/2, 40);

  // Sla elk frame op met genummerde naam: frame-0001.png, frame-0002.png, ...
  saveCanvas('frame-' + nf(frameCount, 4), 'png');
}

De functie nf(getal, cijfers) voegt voorloopnullen toe zodat bestanden correct gesorteerd worden. Zet tijdens het exporteren de frameRate() lager (bv. 5) zodat je computer de tijd heeft om elk frame op te slaan.
