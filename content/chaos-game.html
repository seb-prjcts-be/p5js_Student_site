Het chaos-spel is een van de meest verbluffende demonstraties van generatieve kunst: je volgt een paar heel simpele regels, gooit een dobbelsteen, en er ontstaat vanzelf een perfect fractaal patroon - de Sierpinski-driehoek. Orde uit toeval, in enkele regels code.

Kernpunten
  • Een eenvoudig regelsysteem kan onverwachte, complexe patronen voortbrengen
  • Toeval via floor(random(3)) als digitale dobbelsteen met drie waarden
  • De helft van de afstand berekenen tussen twee punten: (x1 + x2) / 2
  • De Sierpinski-driehoek als spontaan zichtbaar fractaal
  • Varieer het aantal hoekpunten of de sprongverhouding voor andere fractals

Het recept

De regels van het chaos-spel zijn bijna belachelijk eenvoudig:

  1. Kies drie punten op je canvas die samen een driehoek vormen.
  2. Nummer de hoekpunten van 0 tot 2.
  3. Kies een vierde punt P ergens binnen de driehoek.
  4. Maak een digitale dobbelsteen met drie waarden: floor(random(3)).
  5. Gooi de dobbelsteen.
  6. Teken een nieuw punt halverwege P en het geworpen hoekpunt.
  7. Dat nieuwe punt wordt het volgende P. Herhaal duizenden keren.

Wat ontstaat? Geen willekeurige wolk, maar een scherpe, zelfgelijkende fractaal: de Sierpinski-driehoek.

Waarom werkt dit?

Bij elke stap schuif je het punt halverwege naar een hoekpunt op. Dat betekent dat je het punt nooit buiten de driehoek kunt krijgen - de driehoek is een "aantrekker". Binnen die driehoek zijn er regio's die je nooit kunt bereiken door altijd halverwege te springen. Die onbereikbare zones vormen de witte driehoeken van het fractaal.

Weetje

Dit soort systemen - eenvoudige regels die rijke patronen opleveren - heet een Iterated Function System (IFS). Michael Barnsley beschreef ze in 1988 en gebruikte ze om bijvoorbeeld een hyperrealistische varen te genereren met slechts vier transformaties.

In p5.js

let hoeken = [];
let p;

function setup() {
  createCanvas(600, 500);
  background(245);

  // 3 hoekpunten: een driehoek
  hoeken = [
    { x: width / 2, y: 40 },
    { x: 40, y: height - 40 },
    { x: width - 40, y: height - 40 }
  ];

  // Startpunt ergens binnen
  p = { x: width / 2, y: height / 2 };

  stroke(0);
  strokeWeight(1);
}

function draw() {
  // Honderden sprongen per frame
  for (let i = 0; i < 500; i++) {
    let keuze = floor(random(3));       // Dobbelsteen met 3 waarden
    let doel = hoeken[keuze];

    p.x = (p.x + doel.x) / 2;           // Halverwege naar het geworpen hoekpunt
    p.y = (p.y + doel.y) / 2;

    point(p.x, p.y);
  }
}

Merk op dat er niets over fractals in deze code staat. Toch verschijnt de Sierpinski-driehoek vanzelf zodra er genoeg punten zijn getekend.

Variaties

  • Vier hoekpunten (vierkant) met halve-afstandregel: geeft... een vierkant dat bijna vol wordt. Geen fractaal.
  • Vier hoekpunten met de regel "niet twee keer hetzelfde hoekpunt kiezen": nu verschijnt er wel structuur.
  • Vijf hoekpunten met sprongverhouding 0,5: zachte zelfgelijkende vorm.
  • Kleur per hoekpunt: kleur het nieuwe punt volgens de gegooide richting - elk fractaal krijgt zo drie kleurzones.

Probeer variaties. Het maakt pijnlijk duidelijk dat niet elk regelsysteem automatisch een fractaal oplevert - er zit een subtiele balans in.

Oefening

Begin met de basisversie en laat ze minutenlang lopen. Verander daarna één ding tegelijk: de sprongverhouding (probeer 0.4 of 0.618), het aantal hoekpunten, of een kleurcode per dobbelsteenworp. Beschrijf per variant of het beeld meer chaos of meer structuur krijgt.

Experimenteer zelf

Deze versie tekent 500 punten per frame. Wacht een paar seconden en zie de Sierpinski-driehoek verschijnen. Verander de sprongverhouding of voeg een vierde hoekpunt toe.
