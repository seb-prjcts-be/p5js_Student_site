A random walk is a path that emerges step by step from chance. A point chooses a random direction each time and leaves a trail behind.

Why this concept is powerful

The rule is almost ridiculously simple, yet patterns, tangles, and zones of varying density emerge. This makes random walk ideal for thinking about chance and structure.

Think about it: is randomness truly fully chaotic, or does it develop a character after many steps?

What you'll learn here

  • letting a system evolve step by step
  • making randomness visible as a trail
  • working with state: current position and previous position
  • seeing how repetition and time build up an image

The basic rule

At each step you choose a direction and move the point. That's all there is to it:

let x = width / 2;
let y = height / 2;

function step() {
  let choice = floor(random(4));

  if (choice === 0) x += 10;
  if (choice === 1) x -= 10;
  if (choice === 2) y += 10;
  if (choice === 3) y -= 10;
}

By drawing a line after each step, the path becomes visible.

Variations on the same idea

  • biased walk - one direction gets a higher chance
  • self-avoiding walk - the line cannot touch itself
  • multiple walkers - different points build an image together
  • color as memory - each step also changes hue or thickness

This shows that a system often isn't about great complexity, but about a small rule plus a few clever variations.

Exercise

First create a regular random walk. Then add a bias, for example a higher chance of moving right. Compare both trails and describe which image feels more like wandering and which feels more like purposeful movement.
