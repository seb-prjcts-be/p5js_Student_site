Sol LeWitt (1928-2007) was an American conceptual artist who became famous for his Wall Drawings: artworks consisting of written instructions. He sent those instructions to museums, which then had them executed on-site by staff or visitors. The drawing itself was secondary - the instruction was the work.

The instruction as artwork

LeWitt believed that the idea behind an artwork is more important than its execution. His instructions were sometimes one sentence, sometimes several pages. Each execution was different, but the instruction remained the same.

A well-known instruction:

"Lines from corners, sides, and center of the canvas to random points on the wall."

This is nothing other than an algorithm. Code is an instruction. When you write a p5.js sketch, you're doing exactly what LeWitt did: describing a system that someone else - in this case the computer - executes.

Instruction → code

Take this instruction: "Connect 40 randomly placed points with straight lines to their three nearest neighbors."

Translated to code step by step:

// Step 1: generate random points
let points = [];
for (let i = 0; i < 40; i++) {
  points.push({ x: random(width), y: random(height) });
}

// Step 2: for each point - find the 3 nearest
for (let a of points) {
  let sorted = points
    .filter(b => b !== a)
    .sort((b, c) => dist(a.x, a.y, b.x, b.y) - dist(a.x, a.y, c.x, c.y));

  // Step 3: draw a line to each of the 3 nearest neighbors
  for (let i = 0; i < 3; i++) {
    line(a.x, a.y, sorted[i].x, sorted[i].y);
  }
}
