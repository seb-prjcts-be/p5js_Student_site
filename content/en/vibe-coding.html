'Vibe coding' is a way of programming where you describe what you want to make in plain language - and an AI generates the code for you. You don't have to write every line yourself: you describe, test, give feedback, and refine.

What you'll learn
  • How to go from idea to working code in three steps
  • Which free AI tools are suited for which kind of project
  • How to write good prompts for p5.js sketches
  • How to critically evaluate AI-generated code

How does it work?

  1. Describe your idea

    Formulate as concretely as possible what you want to make:

    "Maak een p5.js-sketch met 50 gekleurde cirkels die
van het scherm stuiteren. Elke cirkel heeft een
willekeurige kleur en grootte."

    The more precise your description, the better the result. Think about: what does it do, what does it look like, what happens during interaction?

  2. Let the AI generate

    The AI creates working code based on your description. This is what Copilot generated for the prompt above:

    It works, but it's not perfect yet. The colors are random and the circles pass through each other.

  3. Refine iteratively

    Test the result and give feedback in a follow-up prompt:

    "De cirkels bewegen, maar ik wil dat ze van
elkaar afbotsen in plaats van door elkaar heen
te gaan."

    You repeat this until the result is right. Each round brings it closer to what you had in mind.

Did you know

The term 'vibe coding' was introduced in February 2025 by Andrej Karpathy, co-founder of OpenAI. His original idea: if it works and feels the way you want, you don't have to understand the code line by line.

Free vibe coding tools

There are many AI coding tools, but not all of them are free. These are the best options you can use as a student without paying:

Google AI Studio + Antigravity

Google's all-in-one platform: describe an app and get working code with database and hosting via Firebase. Runs on Gemini.

  • Fully in the browser, no installation needed
  • Can generate complete web apps with backend and database
  • Free limit: ~20 requests per day (significantly reduced since March 2026)
  • aistudio.google.com

Bolt.new

Describe what you want and get a running app directly in the browser. Open source and very fast for prototypes.

  • Generates full-stack apps via WebContainers - everything runs in your browser
  • Open source: you can also run it locally
  • Free limit: 1M tokens/month, 300K/day (~3-8 prompts per day)
  • bolt.new

Replit

Online IDE with AI assistant. Ideal for learning: you can see the code and directly edit and publish it online.

  • Works fully in the browser, including hosting
  • Real-time collaboration possible (like Google Docs)
  • Free limit: limited daily AI credits + 1,200 minutes of dev time per month
  • replit.com

Lovable

Describe an app in plain language and get both code and a nice UI. Strong for visual prototypes and MVPs.

  • Generates code and design at the same time
  • Free projects are publicly visible
  • Free limit: 5 credits per day (30 per month)
  • lovable.dev

v0 (Vercel)

Specialized in generating UI components. Ideal when you need a nice design for a web page.

  • Strong in React/Next.js components with Tailwind CSS
  • Less suited for complete apps, but great for individual parts
  • Free limit: limited number of generations per day
  • v0.dev

GitHub Copilot

AI assistant in your code editor (VS Code). Suggests code as you type and can generate entire functions based on a comment line.

  • Works best when you already have basic coding knowledge
  • Free for students via GitHub Education
  • Integrated in VS Code - no separate tool needed

All tools below have a free version. The limits apply per day or per month.

Vibe coding with p5.js

For p5.js sketches you don't always need a heavy tool. A chatbot like Copilot, duck.ai, ChatGPT or Claude can generate p5.js code just fine - you simply paste the result into the editor on this site or in the online p5.js editor.

Good prompts for p5.js

The more specific your prompt, the better the result. Compare:

// Vaag - AI moet te veel gokken
"Maak iets met cirkels"

// Beter - concreet en visueel
"Maak een p5.js-sketch van 600x400 pixels met
20 cirkels die willekeurig over het canvas bewegen.
Elke cirkel heeft een andere kleur uit een
analoog kleurenschema rond blauw. Als twee cirkels
botsen, wisselen ze van kleur."

Tips for better results

  • Always mention "p5.js" in your prompt - otherwise you sometimes get vanilla Canvas code
  • Include canvas size - "600x400 pixels" prevents guesswork
  • Describe behavior, not just appearance - "bounces off", "follows the mouse", "fades slowly"
  • Ask for explanation - "Add comments that explain what each step does"
  • Iterate step by step - start simple and add features in follow-up prompts

Which tool for p5.js?

  • Copilot / duck.ai / ChatGPT / Claude - the fastest: ask for a sketch, paste the code into the editor. No setup needed. duck.ai works without an account.
  • Replit - handy if you want a full p5.js project with multiple files and online hosting.
  • Bolt.new - if you want to build a complete website around your p5.js sketch (e.g. a portfolio).

Critical evaluation

AI-generated code often works at first glance, but is it actually good? Learning to evaluate what the AI produces is at least as important as being able to generate it.

Watch out

Never paste AI code blindly into an assignment or final project without understanding what it does. Small mistakes (wrong variable names, loops that go one step too far) are easy to miss if you don't read, and you won't learn from them either.

What to look for?

  • Does it work? - Does the code do what you asked? Test thoroughly, not just the happy path.
  • Do you understand it? - Can you explain every line? If not, ask the AI for an explanation or simplification.
  • Is it efficient? - AI sometimes generates unnecessary complexity. Can it be simpler?
  • Are there bugs? - AI makes subtle mistakes: wrong variable names, loops that go one step too far, colors that don't match.
