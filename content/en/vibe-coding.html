'Vibe coding' is a modern approach to programming where you use AI tools to generate code based on your descriptions. Instead of writing every line yourself, you describe what you want to achieve — and the AI turns it into working code.

How does vibe coding work?

The process follows three steps:

1. Describe your idea

Start by clearly formulating what you want to create. For example:

"I want a simple website that shows my portfolio
with a gallery of my digital artworks."

The more specific your description, the better the result. Think about: what does it do, what does it look like, what happens when you click somewhere?

2. Use an AI coding tool

There are various AI-supported environments available:

  • Cursor — An AI-powered code editor that generates and edits code based on plain language. You work in your own project folder.
  • Windsurf — A fork of Visual Studio Code with full support for the AI model of your choice.
  • Replit — An online IDE with an AI assistant that can generate and deploy complete applications based on your description.
  • GitHub Copilot — Integrated in VS Code. Suggests code as you type and can generate entire functions based on a comment line.

An up-to-date and regularly updated overview of AI coding tools can be found at FutureTools.io — filter by the "Coding" category.

3. Iteratively refine

After the AI has generated code, you test it and provide feedback:

"The gallery works, but I want the images
to be displayed in a grid of three columns."

The AI adjusts the code based on your feedback. You repeat this until the result is right.

Advantages

  • Accessibility — Even without deep programming knowledge you can develop functional software.
  • Speed — You can quickly create prototypes and try out ideas without spending much time writing code.
  • Creativity — Because you need to focus less on technical details, you can concentrate on the creative aspect of your project.

Considerations

  • Code quality — AI generates code based on your descriptions, but always check for efficiency and safety. AI makes mistakes.
  • Limited control — You may have less control over the specific implementation, which can be a disadvantage for more complex projects.
  • Dependency — If you don't understand what the AI has written, you'll find it hard to debug or modify when things go wrong.

Did you know?

The term 'vibe coding' was introduced in February 2025 by Andrej Karpathy, co-founder of OpenAI and former AI director at Tesla. He described it as fully going with the flow of AI-driven development — without checking every detail.

This approach enables even beginners to develop software that previously required a whole team of engineers.
