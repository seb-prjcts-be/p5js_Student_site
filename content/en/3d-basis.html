p5.js can do more than just 2D! With the WEBGL renderer you can create 3D graphics: shapes in space, rotations, lighting, and materials.

WEBGL Canvas

To use 3D, pass WEBGL as the third argument:

function setup() {
  createCanvas(600, 400, WEBGL);
}

function draw() {
  background(245);
  box(100);  // Default cube in the center
}

In WEBGL mode, the center of the canvas is at coordinate (0, 0, 0). Use translate() to move objects.

3D Shapes

p5.js has built-in 3D shapes:

  • box(width, height, depth) - cube or box
  • sphere(radius, detail) - sphere
  • cylinder(radius, height) - cylinder
  • cone(radius, height) - cone
  • torus(radius, tubeRadius) - donut shape
  • plane(width, height) - flat surface
function draw() {
  background(30);
  rotateY(frameCount * 0.01);

  // Three shapes side by side
  push();
    translate(-150, 0, 0);
    box(70);
  pop();

  push();
    translate(0, 0, 0);
    sphere(45);
  pop();

  push();
    translate(150, 0, 0);
    torus(40, 15);
  pop();
}

Transformations in 3D

In addition to translate() and scale(), you have three rotation directions:

function draw() {
  background(245);

  rotateX(frameCount * 0.01);  // Tilts forward/backward
  rotateY(frameCount * 0.02);  // Turns left/right
  rotateZ(frameCount * 0.005); // Rolls

  box(100);
}

Always use push() / pop() to isolate transformations:

function draw() {
  background(245);

  push();
    translate(-120, 0, 0);
    rotateY(frameCount * 0.02);
    box(80);
  pop();

  push();
    translate(120, 0, 0);
    rotateX(frameCount * 0.03);
    sphere(50);
  pop();
}

Lighting

Without light, 3D shapes look flat. p5.js has several light sources:

ambientLight() - ambient light

function draw() {
  background(0);
  ambientLight(100);  // Value 0-255: how bright the ambient light is
  sphere(100);
}

directionalLight() - directional light

function draw() {
  background(0);
  // directionalLight(color, direction as vector)
  directionalLight(255, 255, 255, -1, 1, -1);
  sphere(100);
}

pointLight() - point light

function draw() {
  background(0);
  // Light that shines from a single point
  pointLight(255, 200, 100, mouseX - width/2, mouseY - height/2, 200);
  sphere(100);
}

Combination

function draw() {
  background(0);
  ambientLight(40);                             // Weak ambient light
  directionalLight(255, 240, 200, 1, 1, -1);   // Sunlight effect
  pointLight(100, 150, 255, -200, -100, 100);   // Blue accent
  sphere(100);
}

Materials

Materials determine how a shape responds to light:

normalMaterial() - colors based on normal vector

function draw() {
  background(0);
  normalMaterial();  // No extra parameters needed
  box(100);
}

ambientMaterial() - responds to ambient light

function draw() {
  background(0);
  ambientLight(200);
  ambientMaterial(255, 100, 0);  // Orange material
  sphere(100);
}

specularMaterial() - shiny surface

function draw() {
  background(0);
  ambientLight(60);
  directionalLight(255, 255, 255, 0, 0, -1);
  specularMaterial(255);  // Shiny white
  shininess(50);          // How sharp the glint is
  sphere(100);
}

Setting Up the Camera

With camera() you determine where you're looking from:

function draw() {
  background(0);

  // camera(eyeX, eyeY, eyeZ, targetX, targetY, targetZ, upX, upY, upZ)
  camera(0, -200, 400,   // Camera position
         0, 0, 0,        // Point you're looking at
         0, 1, 0);       // "Up" direction

  box(100);
}

// Simpler: orbitControl() for mouse-driven viewing
function draw() {
  background(30);
  orbitControl();  // Click + drag to rotate, scroll to zoom in/out
  box(100);
}

Loading Shaders

Shaders are programs that run directly on the GPU. Load them with await loadShader() in async function setup():

let myShader;

async function setup() {
  createCanvas(600, 400, WEBGL);
  myShader = await loadShader('shader.vert', 'shader.frag');
}

function draw() {
  shader(myShader);
  myShader.setUniform('uTime', millis() / 1000);
  rect(-width / 2, -height / 2, width, height);
}

This is an advanced feature - useful when you want to fully control WEBGL. Shaders are small programs (GLSL) that determine how each pixel looks.
