What if your sketch could see, hear or react to movement - without having to build an AI model yourself? With ml5.js you can load ready-made AI models into p5.js and start creating right away.

What is ml5.js?

ml5.js is a JavaScript library that makes machine learning accessible for makers and designers. It's specifically designed to work with p5.js.

The idea: you don't need to be a mathematician to use AI in your creative work. ml5.js offers pre-trained models you can use directly - for body recognition, sound classification, image recognition and more.

  • Built on top of TensorFlow.js, but much simpler to use
  • Works directly in the browser - no installation needed
  • Open source and free

Loading ml5.js

To use ml5.js in your project, add it via a CDN link. In the live editor on this site you can use it directly in a <script> tag:

<script src="https://unpkg.com/ml5@1/dist/ml5.min.js"></script>

In your own HTML file, place this line in the <head>, next to your p5.js script.

Available models

ml5.js offers several pre-trained models. Here are the main ones:

BodyPose

Recognises the position of the human body via the webcam. Detects points like shoulders, elbows, wrists, knees and ankles. Ideal for interactive installations or movement-driven sketches.

HandPose

Recognises the position of hands and individual fingers. Can detect 21 points per hand. Useful for gesture recognition or drawing with your hands.

FaceMesh

Detects 468 points on the face. With this you can create face filters, analyse emotions or convert facial movements into visual effects.

ImageClassifier

Recognises objects in images. Returns a label ("cat", "car", "flower") with a confidence score. Works with photos and webcam images.

SoundClassifier

Recognises sounds via the microphone. Can distinguish between speaking, clapping, whistling and background noise.

Your first ml5.js sketch: BodyPose

Let's build a sketch step by step that detects your body via the webcam and draws the recognition points.

Step 1: activate webcam

let video;

function setup() {
  createCanvas(640, 480);
  video = createCapture(VIDEO);
  video.size(640, 480);
  video.hide(); // Hide the HTML video element
}

function draw() {
  image(video, 0, 0); // Draw the video on the canvas
}

Step 2: load BodyPose

In modern p5.js, use async function setup() with await, so the model is ready before you continue:

let video;
let bodyPose;
let poses = [];

async function setup() {
  createCanvas(640, 480);
  video = createCapture(VIDEO);
  video.size(640, 480);
  video.hide();

  // Wait until the BodyPose model has loaded
  bodyPose = await ml5.bodyPose();

  // Start detecting poses
  bodyPose.detectStart(video, receiveResults);
}

function receiveResults(results) {
  poses = results;
}

Step 3: draw points

function draw() {
  image(video, 0, 0);

  // Draw all detected points
  for (let i = 0; i < poses.length; i++) {
    let pose = poses[i];
    for (let j = 0; j < pose.keypoints.length; j++) {
      let point = pose.keypoints[j];
      // Only draw if confidence is high enough
      if (point.confidence > 0.2) {
        fill(0, 255, 0);
        noStroke();
        circle(point.x, point.y, 10);
      }
    }
  }
}

Each keypoint has an x, y and confidence value. The confidence indicates how certain the model is of the detection (0 to 1).

From detection to creation

It gets really interesting when you use the detected points to do something visual. Some ideas:

  • Lines between points - Connect the keypoints with lines to draw a skeleton
  • Particles following - Let particles swarm around your hands or head
  • Colour with your body - Use the position of your hands to mix colours
  • Trigger sound - Link movement to sound parameters
  • Abstract shapes - Replace the video image with abstract geometry that follows your body

Important to know

  • Webcam required - Most ml5.js models use the webcam. Make sure your browser has permission.
  • First load takes a moment - The model is downloaded on first use. After that it's faster.
  • Privacy - All processing happens locally in your browser. No images are sent to a server.
  • Performance - AI models in the browser are heavier than regular sketches. On older computers it may run slower.
