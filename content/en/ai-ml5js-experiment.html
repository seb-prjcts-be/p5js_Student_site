You now know the basics of ml5.js. Time to choose your own direction. Which model appeals to you? Which idea do you want to develop? This is the moment to experiment - without a fixed outcome, but with the freedom to make something of your own.

Choose your model

ml5.js offers several models that each process a different type of input. Choose the model that best fits your idea:

Model Input What it does Suitable for
BodyPose Webcam Recognises body points (17 keypoints) Movement interaction, dance, sports
HandPose Webcam Recognises hand points (21 per hand) Gestures, air drawing, control
FaceMesh Webcam Detects 468 facial points Filters, emotion detection, masks
ImageClassifier Webcam / image Recognises objects in images Object recognition, sorting games
SoundClassifier Microphone Recognises sound types Sound-driven art, voice interaction

Ideas for inspiration

Here are some project ideas per model - but the best thing is if you find your own direction.

With BodyPose

  • A digital shadow that follows your movements but reacts with a delay
  • Particles attracted by your hands and repelled by your head
  • An instrument you play by moving - position determines pitch and volume

With HandPose

  • Drawing in the air: your fingertip leaves a trail on the canvas
  • A sign language interpreter: recognise simple hand gestures and show the matching word
  • A virtual puppet that reacts to your finger movements

With FaceMesh

  • A digital mask that moves along with your face
  • Abstract art that reacts to facial expressions - a smile makes colours warmer
  • A mirror that distorts your face based on emotion

With ImageClassifier

  • A game where you hold objects in front of the camera that the model recognises
  • A poetry generator: the model recognises an object and generates a haiku about it

With SoundClassifier

  • A visualisation that reacts differently to speaking, clapping and whistling
  • An interactive installation where silence and sound create two different worlds

Step-by-step plan

A free experiment works best with some structure. Follow this plan:

  1. Choose a model - Which input (camera, microphone) do you want to use?
  2. Start simple - Get the basic detection working first. Draw the raw data on screen.
  3. Add visual layer - Replace the raw points with shapes, colours or patterns that appeal to you.
  4. Make it interactive - Let the visual output react to the AI data in a meaningful way.
  5. Refine and iterate - Adjust colours, timing, size and composition until it feels like yours.

Code template: HandPose

As a starting point for your experiment, you can use this template. It detects your hand and draws circles on the fingertips.

let handPose;
let video;
let hands = [];

async function setup() {
  createCanvas(640, 480);
  video = createCapture(VIDEO);
  video.size(640, 480);
  video.hide();

  // Wait until the HandPose model has loaded
  handPose = await ml5.handPose();
  handPose.detectStart(video, receiveHands);
}

function receiveHands(results) {
  hands = results;
}

function draw() {
  image(video, 0, 0);

  for (let i = 0; i < hands.length; i++) {
    let hand = hands[i];
    for (let j = 0; j < hand.keypoints.length; j++) {
      let point = hand.keypoints[j];
      fill(255, 0, 100);
      noStroke();
      circle(point.x, point.y, 12);
    }
  }
}

Modify this: change the shapes, add trails, experiment with colours that react to the position of your fingers.

Tips for the experiment

  • Test early - Run your code often. Small steps work better than big leaps.
  • Use AI as help - Paste your code into a chatbot and ask: "How do I add a trail effect to these points?"
  • Document your process - Take screenshots or short videos of intermediate results. The process is at least as valuable as the end result.
  • Combine models - Advanced users can use two models simultaneously, for example BodyPose and SoundClassifier.
  • Think about presentation - How do you show this to others? Does it work as an installation? As screen art? As a performance?
