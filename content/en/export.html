Your sketch lives in the browser, but sooner or later you will want to share something: a poster, a GIF for Instagram, a video clip for a project. There are three ways to do that, each with its own use case.

What you will learn
  • Built-in: saveCanvas() for PNG/JPG and saveGif() for short GIF animations
  • Frame-by-frame recording with frameCount and nf() for zero-padding
  • p5.capture: simple UI buttons for video and GIF recording during your sketch
  • p5.export: lossless, frame-perfect MP4 export - even if your sketch runs slowly
  • Which system fits which goal: still image, quick loop, or publication quality

Which system when?

  • One image or a few frames? Use the built-in saveCanvas().
  • Quickly share a GIF or WebM? Use p5.capture with its UI buttons.
  • Lossless MP4 for a publication, Instagram Reel or video project? Use p5.export.

1. Built-in: saveCanvas, saveGif & frames

p5.js has everything on board to save a canvas. No extra library needed.

Static image with saveCanvas()

saveCanvas() downloads the current canvas as PNG or JPG:

function setup() {
  createCanvas(600, 600);
  background(240);
  // ... draw your image ...
  saveCanvas('my-image', 'png');
}

Call saveCanvas() at the end of setup() for a static image, or inside an event like mousePressed() to save on demand:

function mousePressed() {
  saveCanvas('snapshot-' + frameCount, 'png');
}

Frame-by-frame recording

For animation you can save every frame separately and later assemble them into video with tools like ffmpeg or DaVinci Resolve.

Watch out

Lower frameRate() (e.g. 2) before recording every frame. Otherwise your browser writes hundreds of files per second and everything locks up.

The function nf() (number format) turns frameCount into 0001, 0002, ... so your files are in the correct order:

function setup() {
  createCanvas(600, 400);
  frameRate(2);          // slow, so recording can keep up
}

function draw() {
  background(240);
  // ... your animation ...

  // 4 digits, no digits after the decimal: 0001, 0002, ...
  saveCanvas('frame-' + nf(frameCount, 4, 0), 'png');

  if (frameCount >= 120) noLoop();  // stop after 120 frames
}

Result: frame-0001.png, frame-0002.png, ... Later assemble with ffmpeg:

ffmpeg -framerate 30 -i frame-%04d.png -c:v libx264 -pix_fmt yuv420p animation.mp4

Record a GIF with saveGif()

Since p5.js 2.x, saveGif() is built in. You pass a filename and the number of seconds:

function keyPressed() {
  if (key === 'g') {
    saveGif('my-loop', 3);  // record 3 seconds
  }
}

Extra options are available through an options object:

saveGif('my-loop', 3, {
  units: 'seconds',   // or 'frames'
  delay: 0,           // start delay in seconds
  silent: false       // show progress bar
});
Did you know

saveGif() is meant for short loops (a few seconds). For longer or higher-quality videos, p5.capture or p5.export are better.

Strengths & limits

  • No external library needed - works wherever p5.js works.
  • Frame sequence gives maximum quality (every frame is a separate PNG).
  • But: no video container, no audio, and for GIF limited to short clips.

2. p5.capture: video recording with UI buttons

p5.capture is an external library by tapioca24 that adds a small control panel to your sketch: a red record button, a format selector and a progress indicator.

Installation

Add the library to your index.html, after p5.js:

<script src="https://cdn.jsdelivr.net/npm/p5@2/lib/p5.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/p5.capture@1.4.1/dist/p5.capture.min.js"></script>

That is all: the buttons appear automatically in the top-left corner of your sketch.

Supported formats

  • WebM - video for the web (small files, Chrome-friendly)
  • GIF - animation, ideal for social media
  • MP4 - universal video (requires extra encoder)
  • PNG sequence - frames as a ZIP file
  • JPG sequence - frames as a ZIP file
  • WebP - efficient animation container

Programmatic recording

Besides the buttons, you can also start and stop from code:

function setup() {
  createCanvas(600, 600);
}

function keyPressed() {
  if (key === 'r') P5Capture.getInstance().start({
    format: 'gif',
    framerate: 30,
    duration: 180,      // number of frames
    quality: 1.0
  });
  if (key === 's') P5Capture.getInstance().stop();
}

When to use?

  • You want to export a GIF or WebM quickly without installation hassle.
  • Your sketch is real-time and already runs at the desired framerate.
  • For a clip for Instagram Stories, a Discord channel or a quick demo.

For a heavy sketch (many particles, WEBGL) the effective framerate drops and your recording runs slower than intended. For publication quality: use p5.export.

3. p5.export: lossless, frame-perfect MP4

p5.export is a tool by prjcts.be that solves a fundamentally different problem than saveCanvas() or p5.capture: offline encoding. Each frame is fully rendered before it is encoded. The wall-clock time of your browser is thus decoupled from the playback time of the video.

In practice: even if your sketch only hits 5 fps in the browser, you get a smooth 60 fps MP4 video. No frames are lost and no artefacts are added by real-time compression.

Did you know

p5.export is inspired by p5.capture (MIT, tapioca24), but uses Mediabunny + the WebCodecs API for H.264 encoding and the File System Access API to write straight into your project folder. Source: github.com/seb-prjcts-be/p5.export.

What can you do with it?

  • Lossless MP4 (H.264) with guaranteed framerate - no dropped frames
  • PNG sequence for maximum quality (assemble later with ffmpeg)
  • Three profiles: Social (Instagram), Edit master, Lossless
  • Auto save to [projectfolder]/export/YYYYMMDD_HHMMSS.mp4
  • Editor mode: paste an editor.p5js.org URL and export without local install

Two working modes

  1. File System Access (recommended)

    You select your project folder in Chrome or Edge. p5.export gets read/write permission and places your exports directly in an export/ subfolder. No uploads, no downloads - everything local.

  2. Editor mode

    Working online in editor.p5js.org? Paste the public URL of your sketch or download the ZIP and drag it into the drop zone. p5.export loads your project in an iframe, injects a path map for loadImage and loadJSON, and renders.

Workflow

  1. Open p5.export

    Go to prjcts.be/services/p5.export in Chrome or Edge (the File System Access API is required).

  2. Choose your source

    Click Pick folder... for a local folder, or paste an editor.p5js.org URL.

  3. Set your profile

    Choose Social (3.5 Mbps, Instagram-ready), Edit master (20 Mbps, post-production) or Lossless PNG. Set FPS (24/30/60) and duration (seconds).

  4. Record

    Click Record. Each frame is counted at its own pace - not in realtime. Export runs until the desired frame count is reached.

  5. Done

    You get YYYYMMDD_HHMMSS.mp4 in your export/ folder. No further action needed.

Instagram tuning

The Social profile is specifically tuned for Instagram:

  • moov atom at the start of the file (faststart for streaming)
  • mp42 brand (iOS/Instagram recognises this better than default MP4)
  • Keyframe forced every second (better scrubbing and upload)
  • Video-only, no silent audio track (smaller files)
  • Recommended canvas: 1080x1080 (feed) or 1080x1920 (Reels/Stories)

API for custom integration

If you want to call p5.export from your own code, you can use the public API:

const canvas = document.querySelector('canvas');

const { tick, stop } = p5export.startCapture({
  canvas: canvas,
  frames: 300,          // total frame count
  playbackFps: 60,      // output video framerate
  profile: 'social',    // 'social' | 'edit' | 'master'
  format: 'mp4',        // 'mp4' | 'png'
  bitrateMbps: 20,
  onDone: (blob, info) => console.log('done', info),
  onFrame: (n) => console.log('frame', n)
});

// Inside your draw loop, after drawing:
function draw() {
  // ... your sketch ...
  tick();               // count this frame
}

// Manual stop is possible:
// stop();

When to use?

  • You want to publish on Instagram, a festival submission or an archive.
  • Your sketch is too heavy to run in realtime at 60 fps.
  • You want precise control over duration, FPS and bitrate.
  • You want no compression artefacts from real-time encoders.
Watch out

p5.export requires a Chromium browser (Chrome, Edge, Brave) because of the File System Access API. Firefox and Safari do not yet fully support this API.
