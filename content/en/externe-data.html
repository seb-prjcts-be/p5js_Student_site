Data usually lives outside your sketch: in CSV files, JSON files or online via an API. As soon as you pull that data into p5.js, a world of data-driven art opens up: a clock that pulses on live meteorite impacts, a city that changes colour based on temperature, a composition that grows while Wikipedia is being edited.

What you will learn
  • The difference between CSV, JSON and API - and when to use which format
  • Async loading in setup() with await loadTable() and await loadJSON() (p5.js 2.x)
  • Reading data: rows, columns and nested fields
  • Fetching live data from an API with loadJSON()
  • Turning data into image or sound - data as creative material

Over The Edges - data as art

In 2000 the SMAK in Ghent organised the city-wide exhibition Over The Edges. The Milanese artist Alberto Garutti (°1948) had the street lamps on Ghent's Vrijdagmarkt flicker on at every new birth in Ghent. Each baby was a flicker. Years later this work still flickers on in the hearts of many Ghent locals.

That is what data-driven art can be: an invisible stream of numbers - here the birth rates of a city - is made tangible in a material that everyone understands, light. That movement from data to experience is what this chapter is about.

Did you know

Garutti's work is known as "poetic data-art". The most beautiful sonifications and data visualisations do the same: they make you feel what a number alone cannot convey.

CSV - simple tables

CSV stands for Comma-Separated Values. It is probably the simplest file format that exists to store data: rows of values, separated by commas.

name,age,city
Anne,24,Ghent
Frank,31,Brussels
Laila,19,Antwerp

In p5.js 2.x you load a CSV asynchronously in setup():

let table;

async function setup() {
  createCanvas(600, 400);

  // Wait until the file is loaded
  table = await loadTable('data.csv', 'csv', 'header');

  // Access the data
  let rowCount = table.getRowCount();
  let columnCount = table.getColumnCount();

  console.log(rowCount + ' rows, ' + columnCount + ' columns');

  // Loop through all rows
  for (let r = 0; r < rowCount; r++) {
    let name = table.getString(r, 'name');
    let age = table.getNum(r, 'age');
    console.log(name + ' is ' + age + ' years old');
  }
}

The third argument 'header' says that the first row contains column headers. That lets you use names like 'age'; otherwise you have to work with indexes.

JSON - nested data

JSON stands for JavaScript Object Notation. It is a lightweight format that is easy to read and write. Remember the me object from Objects & classes? JSON uses exactly the same notation:

{
  "city": "Ghent",
  "population": 265000,
  "districts": ["Inner city", "Ledeberg", "Gentbrugge"],
  "founded": 630
}

JSON can nest deeper than CSV: arrays inside objects, objects inside arrays. That makes it ideal for richer data like locations with coordinates or games with multiple players.

let data;

async function setup() {
  createCanvas(600, 400);

  data = await loadJSON('city.json');

  // Read a field directly
  console.log(data.city);            // "Ghent"
  console.log(data.districts.length); // 3

  // Loop through an array
  for (let district of data.districts) {
    console.log(district);
  }
}
Tip

Always build up your data reading first with console.log(). Only once you see what is coming in, start drawing. Otherwise you draw into the void.

API - live data from a server

An API (Application Programming Interface) is a set of agreements between software applications. For your sketch that usually means: a URL where you can fetch JSON that is being assembled right now, at this moment.

The function stays the same - loadJSON() - but you pass a URL instead of a local file:

let colors;

async function setup() {
  createCanvas(600, 400);

  // Live data from a public API
  colors = await loadJSON('https://corpora-api.glitch.me/colors');

  noStroke();
  for (let i = 0; i < colors.colors.length; i++) {
    fill(colors.colors[i].rgb);
    rect(i * 60, 0, 60, height);
  }
}
Watch out

Not every API allows requests from a browser. If you see "CORS error" in the console, it means the server does not allow your domain. Find a public API that is browser-friendly instead (see below).

Sources of open data

Here are a few good starting points. Each link is a treasure chest of material for your next sketch:

  • Corpora - hundreds of playful JSON files: animal names, colours, mythology, scents, constellations
  • Corpora API - the same data, but directly accessible via URL
  • Open Data Ghent - mobility, culture, air quality, events
  • Open Data Belgium - thousands of datasets from governments
  • NASA Open Data - space, climate, meteorites
  • NASA Meteorite Landings API - a concrete starting point for data sonification

Creative applications

Now that you know how to load external data, you can start looking for meaning in numbers. A few ideas to get started:

  • A day in your life as data: heart rate, steps, sleep - and put them on screen
  • Air quality from Open Data Ghent mapped to colour or motion
  • The birth rates of your city translated into flickers, like Garutti did
  • A clock built on date and time + live weather data
  • A sonification of CSV numbers from a spreadsheet that means something to you personally

Not every dataset makes good art. Find data that moves you - a neighbourhood, a hobby, a worry - and let something emerge from it.

  1. Loads the file async with await loadJSON() in setup().
  2. Draws the first five items from the list on the canvas as colour fields or text.
  3. Shows a random other item from the list on a mouse click.

Write three sentences about what you noticed: was the data usable as it arrived, or did you have to wrestle with it first?

Conclusion

External data connects your sketch to the world. Whether you load a CSV from your own spreadsheet, get a JSON file from a colleague or use live data from an API: the possibilities are almost endless. The most beautiful work often emerges not by using as much data as possible, but by finding that one meaningful stream and making something tangible from it.
