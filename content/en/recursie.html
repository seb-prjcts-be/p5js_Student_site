Recursion means a function calls itself again. That sounds strange, but it's a powerful way to create branching, fractals, and repetition at multiple scales.

What you will learn
  • Letting a function call itself with a smaller value
  • Writing a clear base case to prevent infinite recursion
  • Drawing at multiple scales with push(), pop(), rotate() and translate()
  • Feeling the difference between a for-loop and a recursive approach
  • Seeing why fractals, trees and lightning often feel recursive

Why this concept matters

Recursion shows that a rich form doesn't always need many different rules. Sometimes a single rule is enough, as long as it's allowed to apply itself again.

Think about it: how can something appear infinitely rich while repeating the same principle over and over?

The base case

A recursive function must know when to stop. Without a base case, you'll keep going forever.

function countDown(n) {
  if (n <= 0) return;
  print(n);
  countDown(n - 1);
}

Here the function calls itself with a smaller value. That's the essence of recursion.

A recursive tree

With a tree, you first draw a trunk. Then from the end you draw two smaller branches. Each branch does the same thing again:

function branch(length) {
  if (length < 12) return;

  line(0, 0, 0, -length);
  translate(0, -length);

  push();
  rotate(0.4);
  branch(length * 0.7);
  pop();

  push();
  rotate(-0.4);
  branch(length * 0.7);
  pop();
}

Recursion versus loops

A for loop is handy when you want to repeat something a fixed number of times. Recursion is powerful when a shape or idea needs to repeat itself at a smaller scale. That's why you often see recursion in trees, rivers, lightning, and fractals.
