p5.js is a JavaScript library that makes creative coding accessible. It's based on Processing, but works entirely in the browser without extra software.

What is Processing?

Processing is a programming language and development environment created in 2001 by Casey Reas and Ben Fry at the MIT Media Lab. The goal was to make programming accessible to artists, designers, and beginners.

Processing uses Java and has its own editor. It's powerful, but requires you to install software.

From Processing to p5.js

p5.js is the JavaScript version of Processing. It was developed by Lauren McCarthy and a community of developers. The key difference:

  • Processing: Java-based, desktop application
  • p5.js: JavaScript-based, works in the browser
  • Processing Python: Python-based, alternative syntax

With p5.js you don't need to install anything - you write code in an editor and open it in your browser!

Processing is free and open source. Anyone can contribute to the software. You can see Processing as an extra layer on top of a programming language - a library of functions that lets you easily create creative projects.

Compare the three languages

The languages are related - you can fairly easily move from one to another:

In this course we focus on p5.js.

The p5.js Editor

The official p5.js editor (editor.p5js.org) is an online development environment where you can start coding right away. Advantages:

  • No installation needed
  • Auto-save
  • Sharing sketches is easy
  • Works on any computer with a browser

You can also create your own HTML file and load p5.js via a CDN (like we do in this course).

The console

Below the code window in the editor you'll see the console. That's your information screen: you can print values to it, and error messages appear there automatically.

// Show a value in the console:
console.log("Hello!");
console.log(mouseX);  // Shows the mouse position

You can also use the console as a calculator: type a calculation and press Enter.

Friendly error messages

The Friendly Error System of p5.js gives understandable error messages in simple language - including a hint about what's wrong and how you can fix it. If there's an error in your code, read the console carefully!

Browser DevTools

Press F12 or Shift + Ctrl + I to open the developer tools. Three useful tabs:

  • Console: error messages and console.log() output appear here. You can also try out JavaScript.
  • Elements: inspect the HTML structure of your page and canvas. Useful to see how p5.js adds elements.
  • Sources: you can set breakpoints here to step through your code line by line when something isn't working.

The Reference

The p5.js Reference is your dictionary for everything p5.js can do. For every function you'll find:

  • Explanation of what a function does
  • What parameters you can use
  • Working examples

Tip: If you don't know how something works, look it up in the Reference. You don't need to memorize everything!

Saving and organizing sketches

In the online editor your sketches are automatically saved when you're logged in. Still, it's worth developing good habits:

  • Use clear names - not "sketch_01" but something like "LES_color_rainbow"
  • No spaces in file names - use an underscore _ or hyphen -
  • Use version numbers when tracking experiments: _v01, _v02
  • Organize in collections - the editor supports folders per project or theme
  • Make regular backups - download the files locally for long-term projects

Project structure

A simple p5.js project looks like this:

my-project/
  ├── index.html
  ├── sketch.js
  └── style.css (optional)

<body>
</body>
</html>

Saving the canvas

With saveCanvas() you save the current canvas as an image (PNG or JPG):

function keyPressed() {
  if (key === 's') {
    saveCanvas('my-sketch', 'png');
  }
}

For an animation you can save each frame with a sequence number:

function draw() {
  background(220);
  circle(frameCount % width, height/2, 40);

  // Save each frame with numbered name: frame-0001.png, frame-0002.png, ...
  saveCanvas('frame-' + nf(frameCount, 4), 'png');
}

The function nf(number, digits) adds leading zeros so files sort correctly. During export, set frameRate() lower (e.g. 5) so your computer has time to save each frame.
