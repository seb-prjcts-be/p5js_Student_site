For a very long time, programmers and designers lived in two very different worlds. They spoke entirely different (computer) languages, dressed differently, and used different computers and operating systems. In 2001 a handful of developers - Casey Reas and Ben Fry - changed this situation for good.

From Processing to p5.js

They launched Processing. It immediately offered countless creative possibilities to artists, designers, hobbyists and students like you. Coding suddenly became easy. Processing is a graphical programming language on top of Java. You need a separate program for it.

The renewed version of Processing, p5.js, developed by Lauren McCarthy, runs on JavaScript (hence the .js!) and works directly in your computer's web browser. It is a very simple programming environment that exists solely to serve your own creativity.

You "sketch" your own ideas in code, fluidly and quickly. Remember the word "sketch" - it captures what we do here remarkably well.

There is also a Python version of Processing.

In this course you will learn about variables, colours, coordinates, loops and randomness. You will intuitively learn to draw points and lines, circles and ellipses, squares and rectangles and much more with your own code. You will learn how to save and reuse your own projects and remix other people's work.

Processing, p5.js and Processing Python

Processing is free and open source. That means there is no copyright on it and anyone can contribute to the software. In principle no one earns money when you use Processing.

You can think of Processing as an extra layer on top of another programming language: a library of new functions that allow you to easily create artistic projects.

Processing as a language is usually very recognisable. The languages Processing works within have their own specifics. The programming languages are related, so you can fairly easily move from one language to another. In this course we focus on p5.js.

Similarities

Don't be discouraged yet, but can you see how Processing is the same across the three languages?

Processing (Java)

void setup() {
  size(400, 400);
}

void draw() {
  background(245);
}

p5.js (JavaScript)

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(245);
}

Processing Python

def setup():
    size(400, 400)

def draw():
    background(245)

Which p5.js version?

This site uses p5.js 2.2.3. The biggest change compared to 1.x: you no longer load assets (images, sound, fonts) in preload(), but with async function setup() and await loadImage(...). You'll see this pattern on the Images, Sound and Text & typography pages.

The official online editor at editor.p5js.org still defaults to p5.js 1.x until August 2026. To work with 2.x there, explicitly pick a 2.x version in the editor, or add ?version=2.2.3 to the URL.

In older tutorials you'll often still see preload(). That code no longer works in 2.x without changes.

The home base: p5js.org

p5js.org is the official home base of the project: documentation, tutorials, examples and access to the online editor.

The p5.js Editor

We chose the p5.js Editor to write our code, but of course there are many other ways to do this.

The editor is an online environment where we can easily code without having to think about servers, browsers, documents and the JavaScript libraries that make programming in p5.js possible. All of those files are kept and organised behind the scenes for us. It is very convenient to just log in directly with your Google account. The first time, click Sign up.

Windows

In the left window we see an area where we can write code directly. Below it we find the console, an area where we can instantly show information - more on that later. The right part of the screen - the preview - is where we see the results of our code come to life.

The console

You'll find the console in the editor below the code. In the example below we calculate a division and show the result at the bottom in the console.

To show a value in the console, use console.log(). You can also use print(), but only inside p5.js, not in plain JavaScript.

// Show a value in the console:
console.log("Hello!");
console.log(mouseX);  // Shows the mouse position

// The console as a calculator:
console.log(10 / 3);

You can think of the console as the screen of your calculator and use it as such. The console is also handy for quickly checking the value of a variable along the way.

Friendly error messages

The Friendly Error System in the p5 editor gives you feedback in the console in a very friendly, smart and supportive way when something goes wrong.

The Friendly Error System is a feature in p5.js specifically designed to provide clear and understandable error messages. Instead of the cryptic errors often encountered in other programming languages, FES tries to describe the error in simple language and offers friendly suggestions for possible solutions.

Did you know

The Friendly Error System is more than just error handling - it is a tool for education and accessibility. By providing users with clear, understandable and actionable feedback, FES lowers the barrier for beginners and makes creative coding accessible to everyone.

The browser developer console

The real developer console of your browser - e.g. Chrome - can be opened by pressing F12 or Shift + Ctrl + I. You'll see how complex a seemingly simple web page can be.

With this developer console you can play safely with the appearance of every page in your browser, because the HTML code has already been downloaded to your computer. You are always looking at a local copy of a web page.

About HTML, CSS and JavaScript

Sketch folder

Our sketch is a folder that we give a name. That folder always contains three files that make up our project. When we use the editor, we can view and edit the three pages separately. Open your files by clicking the > next to sketch.js - the little panel with our files will slide open.

We are really just building a regular web page. The first page (homepage) in a folder on the internet is always called index. That way the browser knows which file to open first. The index.html page is where all the parts come together:

  • The p5.js libraries (p5.js and p5.sound.min.js)
  • The style.css file is responsible for the styling of our page.
  • The sketch.js file where our JavaScript code will live.

Writing your own index.html

A simple p5.js project that you set up yourself looks like this:

my-project/
  ├── index.html
  ├── sketch.js
  └── style.css (optional)
<!DOCTYPE html>
<html>
<head>
  <script src="https://cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.min.js"></script>
  <script src="sketch.js"></script>
</head>
<body>
</body>
</html>
function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(245);
  ellipse(200, 200, 50, 50);
}

What does .min.js mean?

Almost every library comes in two versions:

  • p5.js - the readable version with indentation, comments and clear names. Handy if you want to read or debug the code.
  • p5.min.js - the minified version: all whitespace, comments and long names have been stripped out. Same behaviour, much smaller file.

In production (your actual website) you always use .min.js: faster to load, less data for the visitor. While debugging, the non-minified version can be handier because error messages point to more readable line numbers. For this course and regular projects, .min.js is fine.

This course site itself loads p5.min.js - on the main page and in every live editor on the site. If you right-click → View page source, you'll see the same line as in the example above.

Saving and organising sketches

Good organisation of your p5.js projects is essential to keep your workflow smooth and to give you easy access to your files. Here are some guidelines for managing your projects and sketches.

Use clear and descriptive names

  • Naming: Give your sketches and collections clear and descriptive names. This helps you quickly find specific files and understand what they are for. Because the p5 editor does not currently support visual thumbnails, good naming is crucial.
  • Versioning: Keep multiple versions of your files using version numbers, for example _v01, _v02, and so on. Clearly indicate which version is final.

A handy naming convention looks like this:

  • LES_topic_v01_class_firstname
  • OEF_topic_v01_class_firstname
  • VRIJ_topic_v01_class_firstname

Don't use spaces

File names: Avoid spaces in file names. Use underscores (_) to separate words instead. Spaces in file names can cause unexpected problems when sharing or running your code.

Organise in collections

Organise your sketches in three main collections:

  • LES (lessons) - for sketches made during class.
  • OEF (exercises) - for exercises and assignments.
  • VRIJ (free) - for personal and free experiments.

Share these collections with your teachers so you can easily get feedback and show your progress.

Make regular back-ups

Although the p5.js Web Editor stores your projects online, it is important to make regular back-ups. You can do this easily by downloading your project as a ZIP via File → Download. This protects your work against data loss and lets you keep working offline at any time.

Conclusion

The p5.js Web Editor is a powerful tool for making and sharing interactive projects. By organising your files and collections effectively you can optimise your workflow and keep your projects clear and manageable.
