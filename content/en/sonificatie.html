Sonification is the process of turning data into sound. Where we usually visualise data with charts and colour, sonification lets you hear it. That adds an extra dimension to your p5.js projects, especially when working with external data such as CSV files, JSON or APIs.

What you will learn
  • Mapping data from a file or API onto sound properties (pitch, duration, volume)
  • Working with the p5.Sound library for oscillators and samples
  • Listening to data as a second layer alongside visualisation
  • Example: hearing NASA meteorite data while also plotting it on a map
  • Critical reflection: when does sound add something and when does it get in the way?

What is sonification?

Sonification takes numbers from your data and translates them into audible properties. A row of temperatures can become a melody line where warmer = higher, an earthquake dataset can pulse with intensity, and a long series of sales figures can form a chord that sounds soft or harsh.

The advantage? Our hearing picks up patterns we miss with the eye - rhythm, repetition, sudden spikes. Sound unfolds over time and pulls the listener through the data.

The p5.Sound library

To work with sound in p5.js you need the p5.Sound library. It provides classes for oscillators, samples, filters and analysis.

p5.Sound must be loaded separately alongside p5.js itself:

<script src="https://cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/p5@2.2.3/lib/addons/p5.sound.min.js"></script>
Heads up

Browsers only allow sound to start after a user action (a click or key press). So preferably enable sound inside mousePressed() or with userStartAudio().

Making a data value audible

The smallest building block: an oscillator with a frequency taken from your data.

let osc;

function setup() {
  createCanvas(400, 200);
  osc = new p5.Oscillator('sine');
  osc.amp(0);
  osc.start();
}

function draw() {
  background(245);

  // Map mouseX (0..width) to 200 Hz..800 Hz
  let freq = map(mouseX, 0, width, 200, 800);
  osc.freq(freq);

  text("Frequency: " + round(freq) + " Hz", 20, 30);
}

function mousePressed() {
  osc.amp(0.3, 0.1);  // Fade in
}

function mouseReleased() {
  osc.amp(0, 0.5);    // Fade out
}

The same logic works for any data stream: a value from a CSV column, a JSON field or an API response mapped onto frequency, volume or duration.

Example: listening to meteorites

Seb once made - a long time ago, when speech synthesis was still scratchy - a sonification of meteorite impacts from the NASA Meteorite Landings API. Every landing became a dot on a world map, the location name was spoken by a speech synthesiser, and the mass determined the pitch. Gradually the world map emerged from the data itself.

Did you know

Sonification is also used seriously: NASA runs space and telescope data through sonification software so that blind scientists can "hear" black holes. Try searching for "NASA sonification black hole".

Mapping strategies

The art of sonification lies in the mapping: which data property drives which sound property?

  • Value → pitch - intuitive: large = high, small = low
  • Frequency → rhythm - many data points close together become fast pulses
  • Category → timbre - different data types each get their own instrument
  • Position → stereo panning - the x-axis determines left/right in the sound image
  • Time in data → time in sound - a year = a second, or even a millisecond

Avoid mapping that adds nothing. If your data is already calm, the sound does not need to beep constantly. Sometimes silence is the best sonification of "nothing happened".

Exercise

Pick a small dataset (e.g. hourly temperatures, a row of scores, a week of exchange rates). Choose one number per data point and map it onto the frequency of an oscillator. Play the sequence over five seconds. Describe what you hear that you would not immediately see on a chart.

Conclusion

Sonification is a unique way to explore and understand data. It adds an extra layer of interactivity and experience to your p5.js projects. By combining sound with visual data you build rich, immersive, live experiences. In short: teach your ears to see.
