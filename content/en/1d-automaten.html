In a 1D cellular automaton, everything starts with a single row of cells. Each new row is calculated from the previous one, based on a simple neighbor rule.

Why this concept is powerful

The system is almost minimalist: black or white, on or off. Yet you get patterns that can be textile-like, rhythmic, or surprisingly chaotic.

Think about it: when does simplicity become unpredictable?

What you'll learn here

  • using binary state as a building block
  • deriving new generations from neighbors
  • reading rule numbers like Rule 30 or Rule 110
  • comparing patterns based on their behavior

From three neighbors to a new cell

Each cell looks at three values in the previous row: left, center, and right. This yields 8 possible patterns.

111 110 101 100 011 010 001 000

A rule specifies for each of those patterns whether the new cell becomes 0 or 1. For Rule 30, for example, that's:

00011110

In code

You make a string from the three neighbor cells and check what the rule says. This is Rule 30:

let pattern = "" + left + mid + right;
let newCell = 0;

if (pattern == "111") newCell = 0;
else if (pattern == "110") newCell = 0;
else if (pattern == "101") newCell = 0;
else if (pattern == "100") newCell = 1;
else if (pattern == "011") newCell = 1;
else if (pattern == "010") newCell = 1;
else if (pattern == "001") newCell = 1;
else if (pattern == "000") newCell = 0;

Change the zeros and ones after newCell = and you create your own rule. Each pattern of eight outcomes describes an entire visual language.

Well-known rules

  • Rule 30 - rough and chaotic
  • Rule 90 - symmetric and almost textile-like
  • Rule 110 - complex behavior from very simple rules
  • Rule 184 - sometimes used as a traffic model

Exercise

Compare at least three rules. Give each rule a character description: calm, architectural, rhythmic, chaotic, woven, or aggressive. Which rule would you use for a poster, fabric pattern, or game background?
