An object bundles multiple pieces of data under one name. Each value gets a key (its name) and a value. Perfect for keeping related information together.

What you will learn
  • Creating an object with {} and key/value pairs
  • Reading properties with dot notation: object.property
  • Changing properties and adding new ones
  • Using an object to store things like position, color, and size together

What is an Object?

With an array you store multiple values, but they only have an index (0, 1, 2...). An object gives every value a name:

let person = {
  name: "frank",
  age: 11
};

Between the curly braces you write key: value pairs, separated by commas. The key is the name (e.g. name), the value is what's stored there (e.g. "frank").

Why is it called an "Object Literal"?

Because you write the object out literally between {}. A literal is a value you write straight into the code, as-is:

  • 42 - number literal
  • "hello" - string literal
  • true - boolean literal
  • [1, 2, 3] - array literal
  • { name: "Mila" } - object literal

In short: an object literal is just a bundle of data under one name, written directly in your code.

Reading Properties

Use a dot to fetch a property:

console.log(person.name);  // "frank"
console.log(person.age);   // 11

Changing or Adding Properties

You can change existing values or add new ones:

person.age = 12;              // Change an existing property
person.favouriteColor = "blue";  // Add a new property

console.log(person);
// { name: "frank", age: 12, favouriteColor: "blue" }

Objects in p5.js

In p5 you often use an object to keep the position, size, and color of something together. That saves you from juggling separate variables like ballX, ballY, ballSize:

let ball = {
  x: 300,
  y: 200,
  size: 60,
  color: [100, 150, 250]
};

function draw() {
  background(240);
  fill(ball.color);
  circle(ball.x, ball.y, ball.size);
}

Want dozens of balls, each with their own behaviour? Then a class is the logical next step. That's the next chapter.
