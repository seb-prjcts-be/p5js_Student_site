Langton's Ant is a tiny rule system: an ant moves across a grid and turns left or right depending on the color of the tile beneath it.

Why this concept is so strange

The system starts simple, then becomes messy and chaotic, and after a while it can spontaneously build a kind of highway. This makes Langton's Ant a powerful example of emergence.

Think about it: how can a simple local rule first cause chaos and then order?

What you'll learn here

  • working with grid positions and directions
  • updating a system step by step
  • linking state to behavior
  • seeing how local rules can produce unexpected global behavior

The rules

  • on a white tile: turn right, color the tile black, step forward
  • on a black tile: turn left, color the tile white, step forward

That's all there is to it. Yet the pattern can become very rich.

The structure in code

let cell = grid[x][y];

if (cell === 0) dir = (dir + 1) % 4;
else            dir = (dir + 3) % 4;

grid[x][y] = 1 - cell;
stepForward();

The ant only needs three things: a position, a direction, and a grid with states.

Exercise

First let a regular ant run. Then change the color rules or add a third tile state. Does the system become calmer, wilder, or less interesting?
