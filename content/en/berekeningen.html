p5.js is a calculator that draws. Almost everything you see - positions, sizes, colors, movement - is the result of calculations. How you write those calculations determines what appears on screen.

What you will learn
  • The four basic operators and precedence with parentheses
  • Compound operators (+=, -=, *=, /=) and x++ / x--
  • The modulo operator % for wrapping, rhythm, and grids
  • Random values with random(), and integers via floor(random())
  • Core functions: constrain(), map(), dist(), min(), max(), floor(), round()

The console as a calculator

Before you put calculations into your sketch, you can test them with console.log(). It's the quickest way to check whether an expression does what you expect:

console.log(10 / 3);          // 3.333...
console.log(floor(10 / 3));   // 3
console.log(width / 2);       // Half of the canvas

Open the console in your browser (F12) or in the p5.js editor to see the result. Use it as an exploration tool: try out expressions before putting them in your code.

The four basic operators

let a = 10 + 3;   // 13  addition
let b = 10 - 3;   // 7   subtraction
let c = 10 * 3;   // 30  multiplication
let d = 10 / 4;   // 2.5 division

Order of operations works just like in math: multiplication and division before addition and subtraction. Use parentheses to control the order:

let x = 2 + 3 * 4;    // 14  (not 20)
let y = (2 + 3) * 4;  // 20

Compound operators

Shorthand notations for common operations:

let x = 10;

x += 5;   // x = x + 5  → 15
x -= 3;   // x = x - 3  → 12
x *= 2;   // x = x * 2  → 24
x /= 4;   // x = x / 4  → 6

x++;      // x = x + 1  → 7  (increment by one)
x--;      // x = x - 1  → 6  (decrement by one)

In p5.js you see x += speed and angle += 0.02 everywhere - it's the standard way to make something move or rotate.

The modulo operator: %

% returns the remainder after division. Think of clock arithmetic: after 59 seconds you go back to 0.

7 % 3   // → 1  (7 ÷ 3 = 2, remainder 1)
10 % 5  // → 0  (10 ÷ 5 = 2, remainder 0)
4 % 2   // → 0  (even: remainder is always 0)
5 % 2   // → 1  (odd: remainder is always 1)

Use % to make a value wrap around when it reaches a boundary:

let x = 0;

function draw() {
  x += 3;
  circle(x % width, height / 2, 40);  // wraps automatically at the edge
}

Or to do something at a fixed rhythm:

function draw() {
  // Every 60 frames (≈ every second at 60fps)
  if (frameCount % 60 === 0) {
    background(random(255));
  }

  // Alternating per frame: even or odd
  if (frameCount % 2 === 0) {
    fill(255, 0, 0);
  } else {
    fill(0, 0, 255);
  }
}

And in loops to calculate rows and columns:

for (let i = 0; i < 12; i++) {
  let col = i % 4;        // 0, 1, 2, 3, 0, 1, 2, 3, ...
  let row = floor(i / 4); // 0, 0, 0, 0, 1, 1, 1, 1, ...
  circle(col * 100 + 50, row * 100 + 50, 60);
}

random() - random numbers

random() returns a different decimal number each time. This gives you a different result every time - essential for generative art.

random(100)       // number from 0 to 99.99...  (never exactly 100!)
random(50, 101)   // number from 50 to 100.99...

fill(random(255), random(255), random(255));  // Random color

Need a whole number?

If you want a random integer, use floor() to round down:

// Random number from 0 to 100 (inclusive)
let n = floor(random(101));

// Coin flip: 0 or 1
let side = floor(random(2));  // 0 = heads, 1 = tails
Watch out

random(100) can never return exactly 100, so write floor(random(101)) if you also want to be able to get 100.

Math functions in p5.js

p5.js has a set of built-in functions that save you a lot of writing.

Rounding

round(3.7)   // → 4   (nearest integer)
floor(3.9)   // → 3   (always round down)
ceil(3.1)    // → 4   (always round up)
abs(-5)      // → 5   (absolute value, always positive)

Minimum and maximum

min(3, 7, 1)    // → 1  (smallest value)
max(3, 7, 1)    // → 7  (largest value)

constrain() - keep a value within bounds

Handy to prevent something from going off screen:

let x = constrain(mouseX, 50, width - 50);
// x is never less than 50, never greater than width - 50

map() - translate one range to another

map() is one of the most used functions in generative art. The core idea: you have a value in range A and want to convert it to range B.

// map(value, from_min, from_max, to_min, to_max)

// mouseX (0 to width) → color value (0 to 255)
let red = map(mouseX, 0, width, 0, 255);
fill(red, 0, 150);

// frameCount to an angle (0 to TWO_PI)
let angle = map(frameCount % 120, 0, 120, 0, TWO_PI);

// You can also reverse: big becomes small
let size = map(mouseY, 0, height, 80, 10);  // Top is big, bottom is small
Tip

Almost every interactive or generative effect starts with map(): mouse position to color, distance to size, time to rotation. If you learn one function well, make it this one.

dist() - distance between two points

// Distance from the mouse to the center
let d = dist(mouseX, mouseY, width / 2, height / 2);

// Bigger when the mouse is farther away
let size = map(d, 0, 300, 10, 80);
circle(width / 2, height / 2, size);

sqrt() and sq()

sqrt(16)   // → 4   (square root)
sq(4)      // → 16  (square, same as 4 * 4)
