Imagine: you open a p5.js sketch that someone else wrote. Hundreds of lines of code, no comments. What does this program do? AI can help you decipher unfamiliar code - if you ask the right questions.

Reading code with AI

Understanding code is a skill. Not everything you encounter was written by you, and that's fine. AI can function as a patient explainer that guides you through unfamiliar code.

The key is: ask targeted questions. Instead of "explain this code", try:

  • "What does the draw() function do in this sketch?"
  • "Why is there translate(width/2, height/2) at the beginning?"
  • "Which variables change every frame?"
  • "What would happen if I remove noStroke()?"

The more specific your question, the more useful the answer.

Comparing AI with your own understanding

A powerful exercise: write down first in your own words what you think the code does. Then ask the AI to explain it. Compare the two:

  • Was your assessment correct?
  • Did the AI explain something you overlooked?
  • Did the AI explain something incorrectly? (Yes, that can happen!)

This comparison is where you learn the most. You train yourself to read code and to critically evaluate AI output.

Tips

  • Always copy the complete code to the AI - not just the bit you don't understand. Context matters.
  • Ask follow-up questions if an answer isn't clear: "Can you explain that more simply?"
  • Try to test the explanation: if the AI says "this variable controls the speed", change that variable and see if it's correct.
  • Compare explanations from different AI models - they sometimes explain in very different ways.
