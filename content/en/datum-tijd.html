p5.js has built-in functions to retrieve the current date and time. With these you can build clocks, create countdowns, or tie your sketch to the time of day.

Getting the date and time

Six functions give you the current system time - they are re-evaluated each time you call them:

function draw() {
  let j = year();    // e.g. 2026
  let mo = month();  // 1 – 12
  let d = day();     // 1 – 31

  let u = hour();    // 0 – 23
  let m = minute();  // 0 – 59
  let s = second();  // 0 – 59
}

Combine them for a digital clock:

function draw() {
  background(20);
  fill(255);
  textSize(48);
  textAlign(CENTER, CENTER);

  // Add a leading zero if the number is less than 10
  let u = nf(hour(), 2);
  let m = nf(minute(), 2);
  let s = nf(second(), 2);

  text(u + ":" + m + ":" + s, width / 2, height / 2);
}

nf(value, 2) ensures a fixed width of 2 digits - so 9 is displayed as 09.

Converting time to an angle

An analog clock works by translating seconds, minutes, and hours into angles. Use map() to go from a time range to an angle range.

function setup() {
  createCanvas(400, 400);
  angleMode(DEGREES);
}

function draw() {
  background(20);
  translate(width / 2, height / 2);

  let u = hour() % 12;   // 0 – 11
  let m = minute();       // 0 – 59
  let s = second();       // 0 – 59

  // -90 so that 0 is at the top (12 o'clock) instead of the right
  let sAngle = map(s, 0, 60, 0, 360) - 90;
  let mAngle = map(m, 0, 60, 0, 360) - 90;
  let hAngle = map(u + m / 60, 0, 12, 0, 360) - 90;

  // Second hand
  stroke(255, 60, 60);
  line(0, 0, cos(sAngle) * 130, sin(sAngle) * 130);

  // Minute hand
  stroke(200);
  strokeWeight(3);
  line(0, 0, cos(mAngle) * 100, sin(mAngle) * 100);

  // Hour hand
  stroke(255);
  strokeWeight(5);
  line(0, 0, cos(hAngle) * 65, sin(hAngle) * 65);
}

Smooth second hand

second() jumps one step every second. For smooth motion use millis():

// millis() returns milliseconds - divide by 1000 for seconds
let smoothSec = (millis() / 1000) % 60;
let sAngle = map(smoothSec, 0, 60, 0, 360) - 90;

John Maeda - 12 o'clocks (1996)

In 1996 the American designer and programmer John Maeda created a series of twelve small programs - each a different way to display time. He called them 12 o'clocks.

None of the twelve looks like an ordinary clock. One shows only dots that shift. Another lets concentric circles pulse to the rhythm of the seconds. Yet another draws lines that together express the hour as an abstract pattern.

Maeda's thesis: a clock is an interface for time - and an interface can be reinvented endlessly. The only constraint is that the system must correctly show the current time. How you do that is entirely up to you.

12 o'clocks was one of the earliest examples of code as a design medium. Maeda worked in Java at the time; today you do the same thing in p5.js.

The challenge: make your own clock

Create a sketch that shows the current time - but do it in a way that looks nothing like anything you've seen before.

What you can use:

  • Shapes, lines, points, curves
  • Color, transparency, gradients
  • Movement, rotation, scale
  • Grids, patterns, repetition with for
  • Randomness via random() or noise()
  • Text - with or without numbers

What it must do:

  • Correctly display the current time (hours, minutes, seconds)
  • Live in the loop - animation is not required, but is allowed
  • Show something you would dare to show Maeda

Some ideas to break free from the standard clock:

  • Use the time as a frequency for sin() or cos()
  • Let the number of seconds determine the number of dots or shapes
  • Use color or transparency to encode hours and minutes
  • Build a grid where cells turn on and off based on the time
  • Let the background color slowly shift over the course of the day
