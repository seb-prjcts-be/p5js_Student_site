The chaos game is one of the most astonishing demonstrations of generative art: you follow a few very simple rules, roll a die, and a perfect fractal pattern appears all by itself - the Sierpinski triangle. Order out of randomness, in just a handful of lines of code.

What you will learn
  • A simple rule system can produce unexpected, complex patterns
  • Randomness via floor(random(3)) as a digital die with three values
  • Calculating the halfway point between two points: (x1 + x2) / 2
  • The Sierpinski triangle as a spontaneously visible fractal
  • Varying the number of corners or jump ratio for other fractals

The recipe

The rules of the chaos game are almost absurdly simple:

  1. Pick three points on your canvas that together form a triangle.
  2. Number the corners 0 to 2.
  3. Pick a fourth point P somewhere inside the triangle.
  4. Make a digital die with three values: floor(random(3)).
  5. Roll the die.
  6. Draw a new point halfway between P and the rolled corner.
  7. That new point becomes the next P. Repeat thousands of times.

What appears? Not a random cloud, but a sharp, self-similar fractal: the Sierpinski triangle.

Why does this work?

At every step you move the point halfway towards a corner. That means the point can never escape the triangle - the triangle is an "attractor". Inside that triangle there are regions you can never reach by always jumping halfway. Those unreachable zones form the white triangles of the fractal.

Did you know

This kind of system - simple rules producing rich patterns - is called an Iterated Function System (IFS). Michael Barnsley described them in 1988 and used them to generate a hyperrealistic fern with just four transformations.

In p5.js

let corners = [];
let p;

function setup() {
  createCanvas(600, 500);
  background(245);

  // 3 corner points: a triangle
  corners = [
    { x: width / 2, y: 40 },
    { x: 40, y: height - 40 },
    { x: width - 40, y: height - 40 }
  ];

  // Starting point somewhere inside
  p = { x: width / 2, y: height / 2 };

  stroke(0);
  strokeWeight(1);
}

function draw() {
  // Hundreds of jumps per frame
  for (let i = 0; i < 500; i++) {
    let choice = floor(random(3));      // Die with 3 values
    let target = corners[choice];

    p.x = (p.x + target.x) / 2;         // Halfway to the chosen corner
    p.y = (p.y + target.y) / 2;

    point(p.x, p.y);
  }
}

Notice that nothing in this code mentions fractals. And yet the Sierpinski triangle appears by itself once enough points have been drawn.

Variations

  • Four corners (square) with the halfway rule: gives... a square that fills up almost completely. No fractal.
  • Four corners with the rule "don't pick the same corner twice": now structure does appear.
  • Five corners with jump ratio 0.5: soft self-similar shape.
  • Colour per corner: colour the new point according to the rolled direction - each fractal gets three colour zones.

Try variations. It makes painfully clear that not every rule system automatically produces a fractal - there is a subtle balance at play.

Exercise

Start with the basic version and let it run for several minutes. Then change one thing at a time: the jump ratio (try 0.4 or 0.618), the number of corners, or a colour code per die roll. For each variant, describe whether the image becomes more chaotic or more structured.
