Every shape you draw in p5.js needs coordinates. The canvas has its own coordinate system that by default is slightly different from the one you know from math class.

What you will learn
  • The p5.js coordinate system: (0, 0) is at the top-left
  • X goes to the right, Y goes down - different from math
  • Canvas size defines the boundaries of your coordinates
  • Every point on the canvas has an (x, y) coordinate
  • Experimenting with coordinates in a live editor

Coordinate system

A canvas has a coordinate system that lets you point to any location on the canvas. The origin (0, 0) is at the top-left. X goes to the right, Y goes down.

Did you know

This is different from math where Y goes upward! Many image-processing tools use the top-left origin because screen pixels are also addressed from top to bottom.

From coordinate to shape

Every point on the canvas has an x and y value. If you draw a circle at (200, 200), the centre of that circle sits exactly where x=200 and y=200.

// Point exactly in the centre of a 400x400 canvas:
point(200, 200);

// Circle top-left:
circle(50, 50, 40);

// Circle bottom-right:
circle(350, 350, 40);

Five circles at recognisable positions. Change the numbers to see where the circles appear.

Canvas size and coordinates

The canvas size determines which coordinates you can use. On a 400 x 300 canvas, x runs from 0 to 400 and y from 0 to 300. A point at (500, 100) falls outside the canvas and won't be visible.

Tip

Use width and height instead of literal numbers. That way your code keeps working when you change the canvas size.

function setup() {
  createCanvas(400, 300);

  // Centre, regardless of canvas size:
  circle(width / 2, height / 2, 60);
}
