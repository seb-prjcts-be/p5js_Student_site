Turtle geometry is drawing with an imaginary turtle. You only give simple commands like forward and turn, but from those short instructions complex figures can emerge.

Why this concept is powerful

Turtle geometry makes code very concrete. You're not writing abstract formulas, but steering a drawing body through space. This makes algorithmic thinking feel almost physical.

Think about it: how can a rich drawing emerge from just three or four lines?

What you'll learn here

  • describing space with position and angle
  • building patterns with repetition
  • creating functions that form your own drawing language
  • seeing how a rule not only computes, but also draws

Origin: Logo and Seymour Papert

Turtle graphics became well-known through the programming language Logo. Seymour Papert used the idea to help students understand mathematics through movement and action.

The turtle has three properties:

  • Position - x and y on the canvas
  • Direction - the angle it's facing
  • Pen - down (draws) or up (moves without a line)

The basic commands

forward(n)    // Go n pixels forward and draw a line
right(angle)  // Turn n degrees to the right
left(angle)   // Turn n degrees to the left
penUp()       // Pen up: move without drawing
penDown()     // Pen down: start drawing again

Implementation in p5.js

The turtle keeps track of its position and direction. forward() calculates the new point using sine and cosine:

let tx, ty, heading, penIsDown;

function forward(n) {
  let nx = tx + n * cos(radians(heading));
  let ny = ty + n * sin(radians(heading));
  if (penIsDown) line(tx, ty, nx, ny);
  tx = nx;
  ty = ny;
}

function right(angle) { heading += angle; }
function left(angle)  { heading -= angle; }

From rule to pattern

The power of turtle geometry lies in repetition. A square is nothing more than four times forward and 90 degrees turn.

for (let i = 0; i < 4; i++) {
  forward(100);
  right(90);
}

A five-pointed star uses a different turn angle:

for (let i = 0; i < 5; i++) {
  forward(120);
  right(144);
}

For a regular n-gon you always use 360 / n:

let n = 6;
for (let i = 0; i < n; i++) {
  forward(80);
  right(360 / n);
}

Spirals and deviations

Increase the step length slightly each iteration, and you get a spiral:

for (let i = 0; i < 200; i++) {
  forward(i * 0.5);
  right(91);
}

That small difference between 90 and 91 degrees seems insignificant, but it completely changes the behavior. This is typical for code concepts: small rules can have big visual consequences.

Recursive patterns

If you let the same drawing return at a smaller scale, you end up with recursion and fractals:

function tree(length) {
  if (length < 5) return;
  forward(length);
  right(30);
  tree(length * 0.7);
  left(60);
  tree(length * 0.7);
  right(30);
  forward(-length);
}

tree(80);

Here you can see that turtle geometry is not just about shapes, but also about structure and growth.

Exercise

Make three versions of the same idea:

  • a neat shape with a "correct" angle, like 90 or 120 degrees
  • a version with a small deviation, like 89 or 121 degrees
  • a personal variant with color, variable step length, or pen-up movements

Then compare which pattern feels the most predictable and which one becomes surprisingly different.
