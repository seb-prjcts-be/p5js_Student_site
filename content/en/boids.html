Boids are simple agents that together form flocking behavior. Each boid follows a few rules, but together they seem to move almost intelligently.

What you will learn
  • Working with agents that each keep their own position, velocity and acceleration
  • Using vectors (p5.Vector) for direction and speed
  • The three classic rules: alignment, cohesion and separation
  • Seeing how local interaction between neighbours produces a group pattern without a leader
  • Balancing attraction, repulsion and alignment with limit() and setMag()

Why this concept is fascinating

Boids show that a group doesn't need a leader to appear coordinated. The order doesn't come from a central plan, but from the interaction between neighbors.

Think about it: do you need a leader to get collective behavior?

The three classic rules

  • alignment - move roughly in the same direction as your neighbors
  • cohesion - stay close to the group
  • separation - don't bump into each other

These three rules together create flocking: behavior that resembles birds, fish, or a swarm of drones.

In code

Each boid calculates forces based on nearby boids:

for (let other of boids) {
  let d = dist(this.x, this.y, other.x, other.y);

  if (d < visionRadius) {
    alignment.add(other.vel);
    cohesion.add(other.pos);
  }

  if (d > 0 && d < collisionRadius) {
    separation.add(differenceVector);
  }
}

Then those forces are limited and added to the velocity.

Exercise

Build separation first. Then add alignment, and only then cohesion. This way you really feel what effect each rule has. Which combination looks calm, which one looks nervous?
