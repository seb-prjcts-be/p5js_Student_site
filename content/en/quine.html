A quine is a program that produces its own source code without opening a file. It's a small, strange, and surprisingly deep idea about self-reference in code.

Why this concept is special

Most code does something outside of itself: drawing, calculating, reacting, or simulating. A quine turns that around. The program itself becomes the subject.

Think about it: can a system describe itself without getting help from the outside?

What you'll learn here

  • the difference between code and data
  • how strings can contain a description of code
  • what self-reference means in computer science
  • why a clever idea is sometimes more important than lots of code

Self-reference

The core problem: how do you write a program that describes itself without simply reading a file? You can't use readFile() or a network. So the code must carry itself as data.

The trick is quoting: you store part of the code in a string and use that string to reconstruct the complete source code.

// Pseudocode quine
let s = "let s = X; print('let s = ' + quote(s) + '; ' + s)";
print("let s = " + quote(s) + "; " + s);

An accessible version in JavaScript

In JavaScript every function has a built-in way to read its own source code: function.toString().

function setup() {
  let code = setup.toString();
  console.log(code);
}

This isn't a perfect quine, because you're only reading the function and not the complete program. But didactically it's a strong stepping stone.

Self-knowledge on the canvas

In p5.js you can have a function literally draw itself:

function setup() {
  createCanvas(600, 400);
  background(20);

  let code = setup.toString();
  let lines = code.split("\n");

  fill(140, 230, 120);
  textFont("monospace");
  textSize(11);

  lines.forEach(function(line, i) {
    text(line, 10, 20 + i * 15);
  });
}

Why this is more than a joke

A quine opens a door to deeper questions: what is a description, what is a representation, and where is the boundary between system and meta-system? That's why this topic belongs among creative code concepts: it's code as philosophy.

Real quines

A full quine in JavaScript looks more complicated. You don't need to memorize it. What matters is that you understand why it works:

(function q(){
  let s = "(function q(){\n  let s = " +
          JSON.stringify(s) + ";\n  " + q.toString().split("\n").slice(2,-1).join("\n  ") +
          "\n})();";
  console.log(s);
})();

Quines exist in almost every programming language. They show that code can not only execute itself, but also represent itself.
