Bij een 1D cellulair automaat begint alles met een enkele rij cellen. Elke nieuwe rij wordt berekend uit de vorige, op basis van een simpele buurregel.

Waarom dit concept sterk is

Het systeem is bijna minimalistisch: zwart of wit, aan of uit. Toch krijg je patronen die textielachtig, ritmisch of verrassend chaotisch kunnen worden.

Denkvraag: wanneer wordt eenvoud onvoorspelbaar?

Wat je hier leert

  • binaire toestand gebruiken als bouwsteen
  • nieuwe generaties afleiden uit buren
  • regelnummers zoals Rule 30 of Rule 110 lezen
  • patronen vergelijken op basis van hun gedrag

Van drie buren naar een nieuwe cel

Elke cel kijkt naar drie waarden in de vorige rij: links, midden en rechts. Dat levert 8 mogelijke patronen op.

111 110 101 100 011 010 001 000

Een regel zegt voor elk van die patronen of de nieuwe cel 0 of 1 wordt. Bij Rule 30 is dat bijvoorbeeld:

00011110

In code

Je maakt een string van de drie buurcellen en kijkt wat de regel zegt. Dit is Rule 30:

let patroon = "" + left + mid + right;
let nieuweCel = 0;

if (patroon == "111") nieuweCel = 0;
else if (patroon == "110") nieuweCel = 0;
else if (patroon == "101") nieuweCel = 0;
else if (patroon == "100") nieuweCel = 1;
else if (patroon == "011") nieuweCel = 1;
else if (patroon == "010") nieuweCel = 1;
else if (patroon == "001") nieuweCel = 1;
else if (patroon == "000") nieuweCel = 0;

Verander de nullen en enen achter nieuweCel = en je maakt je eigen regel. Zo beschrijft elk patroon van acht uitkomsten een volledige beeldtaal.

Bekende regels

  • Rule 30 - ruig en chaotisch
  • Rule 90 - symmetrisch en bijna textielachtig
  • Rule 110 - complex gedrag uit heel simpele regels
  • Rule 184 - wordt soms gebruikt als verkeersmodel

Oefening

Vergelijk minstens drie regels. Geef elke regel een karakteromschrijving: rustig, architecturaal, ritmisch, chaotisch, geweven of agressief. Welke regel zou je gebruiken voor een poster, stofpatroon of game-achtergrond?
