Stel je voor: je opent een p5.js-sketch die iemand anders heeft geschreven. Honderden regels code, geen commentaar. Wat doet dit programma? AI kan je helpen om onbekende code te ontcijferen - als je de juiste vragen stelt.

Kernpunten
  • Gerichte vragen stellen in plaats van "leg deze code uit"
  • Een mystery sketch lezen en voorspellen wat er op het scherm verschijnt
  • Vier standaardvragen: wat doet het, regel voor regel, welke p5.js-functies, hoe aanpassen
  • Eigen inschatting eerst opschrijven en daarna vergelijken met de AI-uitleg
  • Uitleg testen door code aan te passen en te controleren of het klopt

Code lezen met AI

Code begrijpen is een vaardigheid. Niet alles wat je tegenkomt heb je zelf geschreven, en dat is prima. AI kan functioneren als een geduldige uitlegger die je door onbekende code loodst.

De sleutel is: stel gerichte vragen. In plaats van "leg deze code uit", probeer:

  • "Wat doet de draw()-functie in deze sketch?"
  • "Waarom staat er translate(width/2, height/2) aan het begin?"
  • "Welke variabelen veranderen elke frame?"
  • "Wat zou er gebeuren als ik noStroke() weghaal?"

Hoe specifieker je vraag, hoe bruikbaarder het antwoord.

Oefening: mystery sketch

Hieronder staat een sketch die je waarschijnlijk niet meteen begrijpt. Bestudeer de code en probeer te voorspellen wat er op het scherm verschijnt. Gebruik daarna een AI-tool om je vragen te stellen.

Mystery sketch

Wat je kunt vragen

Kopieer de code uit de editor hieronder en plak hem in een AI-tool. Probeer deze vragen:

  1. "Wat doet deze code?" - Vraag om een samenvatting in gewone taal.
  2. "Leg regel voor regel uit wat er gebeurt." - Voor als je elk detail wilt snappen.
  3. "Welke p5.js-functies worden hier gebruikt en wat doen ze?" - Om je woordenschat uit te breiden.
  4. "Hoe zou ik deze sketch kunnen aanpassen om [X] te doen?" - Om van begrip naar creatie te gaan.

Probeer het zelf

Hier is dezelfde mystery sketch in de editor. Pas de code aan en kijk wat er verandert. Kun je voorspellen wat elke aanpassing doet?

AI vergelijken met je eigen begrip

Een krachtige oefening: schrijf eerst in je eigen woorden op wat je denkt dat de code doet. Vraag daarna aan de AI om het uit te leggen. Vergelijk de twee:

  • Klopte jouw inschatting?
  • Heeft de AI iets uitgelegd dat je over het hoofd zag?
  • Heeft de AI iets fout uitgelegd? (Ja, dat kan!)

Dit vergelijken is waar je het meest van leert. Je traint jezelf om code te lezen én om AI-output kritisch te evalueren.

Tips

  • Kopieer altijd de volledige code naar de AI - niet alleen het stukje dat je niet snapt. Context is belangrijk.
  • Vraag door als een antwoord niet duidelijk is: "Kun je dat eenvoudiger uitleggen?"
  • Probeer de uitleg te testen: als de AI zegt "deze variabele bepaalt de snelheid", verander die variabele dan en kijk of het klopt.
  • Vergelijk uitleg van verschillende AI-modellen - ze leggen soms op heel andere manieren uit.
