'Vibe coding' is een manier van programmeren waarbij je in gewone taal beschrijft wat je wilt maken - en een AI genereert de code voor je. Je hoeft niet elke regel zelf te schrijven: je beschrijft, test, geeft feedback, en verfijnt.

Wat je leert
  • Hoe je van idee naar werkende code komt in drie stappen
  • Welke gratis AI-tools geschikt zijn voor welk soort project
  • Hoe je goede prompts schrijft voor p5.js-sketches
  • Hoe je AI-gegenereerde code kritisch beoordeelt

Hoe werkt het?

  1. Beschrijf je idee

    Formuleer zo concreet mogelijk wat je wilt maken:

    "Maak een p5.js-sketch met 50 gekleurde cirkels die
van het scherm stuiteren. Elke cirkel heeft een
willekeurige kleur en grootte."

    Hoe preciezer je beschrijving, hoe beter het resultaat. Denk aan: wat doet het, hoe ziet het eruit, wat gebeurt er bij interactie?

  2. Laat de AI genereren

    De AI maakt werkende code op basis van je beschrijving. Dit is wat Copilot genereerde voor de prompt hierboven:

    Werkt, maar nog niet perfect. De kleuren zijn willekeurig en de cirkels gaan door elkaar heen.

  3. Verfijn iteratief

    Test het resultaat en geef feedback in een vervolgprompt:

    "De cirkels bewegen, maar ik wil dat ze van
elkaar afbotsen in plaats van door elkaar heen
te gaan."

    Dit herhaal je tot het resultaat klopt. Elke ronde breng je het dichter bij wat je voor ogen hebt.

Weetje

De term 'vibe coding' werd in februari 2025 geïntroduceerd door Andrej Karpathy, mede-oprichter van OpenAI. Zijn originele idee: als het werkt en voelt zoals je wilt, hoef je de code niet regel voor regel te begrijpen.

Gratis vibe coding tools

Er zijn veel AI-codeertools, maar niet allemaal gratis. Dit zijn de beste opties waar je als student zonder te betalen mee aan de slag kunt:

Google AI Studio + Antigravity

Google's alles-in-één platform: beschrijf een app en krijg werkende code met database en hosting via Firebase. Draait op Gemini.

  • Volledig in de browser, geen installatie nodig
  • Kan volledige webapps genereren met backend en database
  • Gratis limiet: ~20 requests per dag (sterk verlaagd sinds maart 2026)
  • aistudio.google.com

Bolt.new

Beschrijf wat je wilt en krijg een draaiende app direct in de browser. Open source en heel snel voor prototypes.

  • Genereert full-stack apps via WebContainers - alles draait in je browser
  • Open source: je kunt het ook lokaal draaien
  • Gratis limiet: 1M tokens/maand, 300K/dag (~3–8 prompts per dag)
  • bolt.new

Replit

Online IDE met AI-assistent. Ideaal om te leren: je ziet de code én kunt die direct aanpassen en online zetten.

  • Werkt volledig in de browser, inclusief hosting
  • Realtime samenwerken mogelijk (zoals Google Docs)
  • Gratis limiet: beperkte dagelijkse AI-credits + 1.200 minuten dev-tijd per maand
  • replit.com

Lovable

Beschrijf een app in gewone taal en krijg zowel code als een mooie UI. Sterk voor visuele prototypes en MVP's.

  • Genereert code én design tegelijk
  • Gratis projecten zijn publiek zichtbaar
  • Gratis limiet: 5 credits per dag (30 per maand)
  • lovable.dev

v0 (Vercel)

Gespecialiseerd in het genereren van UI-componenten. Ideaal als je een mooi design nodig hebt voor een webpagina.

  • Sterk in React/Next.js-componenten met Tailwind CSS
  • Minder geschikt voor volledige apps, wél voor losse onderdelen
  • Gratis limiet: beperkt aantal generaties per dag
  • v0.dev

GitHub Copilot

AI-assistent in je code-editor (VS Code). Suggereert code terwijl je typt en kan hele functies genereren op basis van een commentaarregel.

  • Werkt het best als je al basiskennis van code hebt
  • Gratis voor studenten via GitHub Education
  • Geïntegreerd in VS Code - geen aparte tool nodig

Alle tools hieronder hebben een gratis versie. De limieten gelden per dag of per maand.

Vibe coding met p5.js

Voor p5.js-sketches heb je niet altijd een zware tool nodig. Een chatbot zoals Copilot, duck.ai, ChatGPT of Claude kan prima p5.js-code genereren - je plakt het resultaat gewoon in de editor op deze site of in de online p5.js-editor.

Goede prompts voor p5.js

Hoe specifieker je prompt, hoe beter het resultaat. Vergelijk:

// Vaag - AI moet te veel gokken
"Maak iets met cirkels"

// Beter - concreet en visueel
"Maak een p5.js-sketch van 600x400 pixels met
20 cirkels die willekeurig over het canvas bewegen.
Elke cirkel heeft een andere kleur uit een
analoog kleurenschema rond blauw. Als twee cirkels
botsen, wisselen ze van kleur."

Tips voor betere resultaten

  • Noem altijd "p5.js" in je prompt - anders krijg je soms vanilla Canvas-code
  • Geef canvasgrootte mee - "600x400 pixels" voorkomt gokwerk
  • Beschrijf gedrag, niet alleen uiterlijk - "stuitert af", "volgt de muis", "vervaagt langzaam"
  • Vraag om uitleg - "Voeg Nederlandse commentaren toe die uitleggen wat elke stap doet"
  • Itereer stap voor stap - begin simpel en voeg features toe in volgende prompts

Voorbeeld: één prompt, werkend resultaat

Deze volledige sketch is gegenereerd met één enkele prompt: "Maak een p5.js-sketch met deeltjes die de muis volgen en een kleurenspoor achterlaten."

Beweeg je muis over het canvas. Merk op: de AI heeft zelf gekozen voor een vervagend spoor, kleur op basis van muispositie en een levensduur-systeem. Kun je het verbeteren? Probeer zwaartekracht toe te voegen of de kleuren aan te passen.

Welke tool voor p5.js?

  • Copilot / duck.ai / ChatGPT / Claude - het snelst: vraag om een sketch, plak de code in de editor. Geen setup nodig. duck.ai werkt zonder account.
  • Replit - handig als je een volledig p5.js-project wilt met meerdere bestanden en online hosting.
  • Bolt.new - als je een complete website wilt bouwen rondom je p5.js-sketch (bv. een portfolio).

Kritisch evalueren

AI-gegenereerde code werkt vaak op het eerste gezicht, maar is het ook goed? Leren beoordelen wat de AI oplevert is minstens zo belangrijk als het kunnen genereren.

Let op

Plak nooit blind AI-code in een opdracht of eindproject zonder te begrijpen wat er staat. Kleine fouten (verkeerde variabelenamen, loops die net te ver gaan) zijn makkelijk te missen als je niet leest, en je leert er ook niets van.

Waar let je op?

  • Werkt het? - Doet de code wat je hebt gevraagd? Test grondig, niet alleen het happy path.
  • Begrijp je het? - Kun je elke regel uitleggen? Als niet, vraag de AI om uitleg of vereenvoudiging.
  • Is het efficiënt? - AI genereert soms onnodige complexiteit. Kan het eenvoudiger?
  • Zijn er fouten? - AI maakt subtiele fouten: verkeerde variabelenamen, loops die net te ver gaan, kleuren die niet kloppen.

Oefening: beoordeel deze sketch

Stel dat je een AI hebt gevraagd: "Maak een p5.js-sketch met vallende sneeuwvlokken." Je krijgt de code hieronder. Beoordeel: wat is goed? Wat ontbreekt? Wat zou je verbeteren?

Denk na over: is dit overtuigend als sneeuw? Wat mist er? Misschien horizontale beweging, variatie in transparantie, of een zachter kleurenpalet? Probeer de code te verbeteren.
