'Vibe coding' is een moderne benadering van programmeren waarbij je AI-tools gebruikt om code te genereren op basis van jouw beschrijvingen. In plaats van zelf elke regel te schrijven, beschrijf je wat je wilt bereiken — en de AI zet dit om in werkende code.

Hoe werkt vibe coding?

Het proces verloopt in drie stappen:

1. Beschrijf je idee

Begin met het duidelijk formuleren van wat je wilt maken. Bijvoorbeeld:

"Ik wil een eenvoudige website die mijn portfolio toont
met een galerij van mijn digitale kunstwerken."

Hoe concreter je beschrijving, hoe beter het resultaat. Denk aan: wat doet het, hoe ziet het eruit, wat gebeurt er als je ergens op klikt?

2. Gebruik een AI-codingtool

Er zijn verschillende AI-ondersteunde omgevingen beschikbaar:

  • Cursor — Een AI-aangedreven code-editor die code genereert en bewerkt op basis van gewone taal. Je werkt in je eigen projectmap.
  • Windsurf — Een fork van Visual Studio Code met volledige ondersteuning voor het AI-model van jouw keuze.
  • Replit — Een online IDE met een AI-assistent die volledige applicaties kan genereren en online zetten op basis van je beschrijving.
  • GitHub Copilot — Geïntegreerd in VS Code. Suggereert code terwijl je typt en kan hele functies genereren op basis van een commentaarregel.

Een actueel en regelmatig bijgewerkt overzicht van AI-codeertools vind je op FutureTools.io — filter op de categorie "Coding".

3. Iteratief verfijnen

Nadat de AI code heeft gegenereerd, test je die en geef je feedback:

"De galerij werkt, maar ik wil dat de afbeeldingen
in een raster van drie kolommen worden weergegeven."

De AI past de code aan op basis van je feedback. Dit herhaal je totdat het resultaat klopt.

Voordelen

  • Toegankelijkheid — Zelfs zonder diepgaande programmeerkennis kun je functionele software ontwikkelen.
  • Snelheid — Je kunt snel prototypes maken en ideeën uitproberen zonder veel tijd te besteden aan het schrijven van code.
  • Creativiteit — Doordat je je minder hoeft te concentreren op technische details, kun je je richten op het creatieve aspect van je project.

Overwegingen

  • Kwaliteit van de code — AI genereert code op basis van je beschrijvingen, maar controleer altijd op efficiëntie en veiligheid. AI maakt fouten.
  • Beperkte controle — Je hebt mogelijk minder greep op de specifieke implementatie, wat een nadeel kan zijn bij complexere projecten.
  • Afhankelijkheid — Als je niet begrijpt wat de AI heeft geschreven, kun je het moeilijk debuggen of aanpassen wanneer het misloopt.

Wist je dat?

De term 'vibe coding' werd in februari 2025 geïntroduceerd door Andrej Karpathy, mede-oprichter van OpenAI en voormalig AI-directeur bij Tesla. Hij beschreef het als volledig meegaan in de flow van AI-gestuurde ontwikkeling — zonder elk detail te controleren.

Deze benadering stelt zelfs beginners in staat om software te ontwikkelen die vroeger een heel team van ingenieurs vereiste.
