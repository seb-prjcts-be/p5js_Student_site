Functies, objecten en klassen

Functies

Maak je code herbruikbaar en beter georganiseerd.

Inleiding

Nu je hebt geleerd hoe je met variabelen en DOM-elementen kunt werken, is het tijd om je code naar een hoger niveau te tillen door je eigen functies te schrijven. In dit hoofdstuk leer je hoe je een eigen functie kunt maken, zoals de veranderKleur()- functie die we eerder hebben genoemd.

Wat is een functie?

Een functie is een blok code, afgebakend door accolades, die een specifieke taak uitvoert.
Je kunt het zien als een mini-programma binnen je hoofdprogramma.
Het mooie van functies is dat je ze steeds opnieuw kunt gebruiken zonder de code te herhalen.

We zijn al bekend met de twee functies setup() en draw(), maar we kunnen ze dus zelf ook  schrijven.

De basis van een functie

Het schrijven van een functie begint met het function- sleutelwoord, gevolgd door de naam van de functie en haakjes.

function veranderKleur() {

    // Code om de kleur te veranderen bvb.

    knop.style('background-color', '❚lime');

}

Waarom een functie gebruiken?

Het schrijven van je eigen functies maakt je code niet alleen herbruikbaar maar ook veel makkelijker te lezen en te onderhouden.
Je kunt complexe taken opsplitsen in kleinere, beheersbare functies.

Een functie aanroepen

function setup() {

  createCanvas(400, 400);

  background(220);

}

function draw() {

  if (frameCount % 50 == 0)

  {

       veranderKleur();

  }

}

function veranderKleur(){

  background(random(255),random(255),random(255));

}

Conclusie

Het kunnen schrijven van je eigen functies is een cruciale vaardigheid voor elke programmeur. Functies maken je code georganiseerd, herbruikbaar en makkelijk te begrijpen. Of je nu een beginner bent of een ervaren ontwikkelaar, het effectief gebruik van functies zal je helpen om meer efficiënte en krachtige programma's te schrijven.

Doe meer met je functies

Parameters meegeven

Soms wil je dat je functie flexibel is en verschillende waarden kan verwerken. Dat is waar parameters van pas komen. Je kunt parameters zien als de ingrediënten die je aan een functie geeft om mee te werken.

In dit voorbeeld neemt de functie berekenDeSom() twee parameters, a en b, en print hun som.
Je kunt duizenden verschillende waarden voor a en b invoeren, waardoor de functie heel erg  veelzijdig en herbruikbaar wordt.

berekenDeSom(81,37);

…

function berekenDeSom(a,b) {

    print(a+b);

}


of


Return, de uitkomst van je functie

Soms wil je dat een functie een waarde teruggeeft die je later in je code kunt gebruiken.
Dit doe je met het return-sleutelwoord.

print(berekenDeSom(31,92));

…

function berekenDeSom(a,b) {

    return(a+b);

}

Conclusie

Parameters en return zijn krachtige concepten die je functies flexibel en herbruikbaar maken.

Met parameters kun je variabele ingrediënten aan je functies toevoegen, en met return kun je de uitkomst van een functie teruggeven. Door deze concepten te begrijpen en toe te passen, kun je complexere en dynamische programma's schrijven.

Objecten

Inleiding

Nu je de basis van variabelen, functies en DOM-elementen onder de knie hebt, is het tijd om een stap verder te gaan. In dit hoofdstuk leer je hoe je met enkele objecten kunt werken om complexere structuren in je code te creëren.

Wat is een object?

Een object is een verzameling van gerelateerde variabelen (eigenschappen) ,functies (methodes) en gebeurtenissen (events) gebundeld onder één naam. Je kunt het zien als een mini-database waarin je verschillende soorten informatie kunt opslaan en bewerken.

Jij, als object met eigenschappen

Stel dat je jezelf als een object zou willen opslaan, dan hebben een hele hoop gegevens nodig:
naam en voornaam; geboortedatum, woonplaats, hobby’s, en nog zoveel meer…

let ikke={

    voornaam:"Frank",
   naam:"Van de Velde",

    geboortedatum: "1 januari 2000",

    hobby:"paardrijden"
}

Let op de vreemde schrijfwijze!
Dubbele punten, komma’s en geen komma na het laatste item!

Het “ikke”-object gebruiken

let ikke={

    voornaam:"Frank",

    naam:"Van de Velde",

    geboortedatum: "1 januari 2000",

    hobby:"paardrijden"

}

   // Let op de "dot notation"

   console.log(ikke.voornaam);

   console.log(ikke.naam);

   console.log(ikke.geboortedatum);

   console.log(ikke.hobby);

   // Frank doet niet meer een paardrijden, maar aan schaken…
  ikke.hobby="schaken";
  // Woonplaats toevoegen aan het object ikke

   ikke.stad="Gent";

Conclusie

Werken met enkele objecten is een geweldige manier om je code te organiseren en voor te bereiden op meer complexe structuren. Objecten bieden een manier om gerelateerde variabelen te groeperen, wat je code leesbaarder en onderhoudsvriendelijker maakt.

Let op! Als je meerdere objecten wil gebruiken, dan hebben we een ander, maar erg krachtig systeem nodig: de klasse.

Klassen

Inleiding

Na het begrijpen van enkele objecten is de volgende logische stap het leren over klassen.
Pas op, want dat is een grote stap!  Een klasse is als een blauwdruk voor het maken van heel veel verschillende objecten en biedt een gestructureerde manier om code te organiseren.
We moeten onze kennis van arrays zeker weer bovenhalen.

Wat?

Een klasse is een sjabloon dat definieert hoe objecten die ervan zijn gemaakt, eruitzien zullen zien en zich zullen gedragen. Het is een manier om meerdere objecten met vergelijkbare eigenschappen en gedragingen te creëren.

Meerdere? Honderden! Duizenden!

Naast het opslaan van informatie in de vorm van eigenschappen, kunnen objecten ook functies bevatten, die we methodes noemen. Methodes zijn de acties die een object kan uitvoeren. Ze zijn in wezen functies die zijn ingebed in een object en meestal werken met de eigenschappen van dat object.

In dit voorbeeld heeft het bal object twee methodes: bewegen() en tonen().

  • Bewegen():  veranderen de x en y coördinaten van elke walker op een willekeurige manier.
  • Tonen(): de walker op het scherm tekenen met behulp van de ellipse functie.
  • Beide methodes gebruiken het this sleutelwoord om toegang te krijgen tot de eigenschappen van het object zelf.

Methodes maken je objecten dynamisch en interactief. Ze stellen je in staat om complex gedrag in te kapselen in een enkel object, waardoor je code georganiseerd en gemakkelijk te begrijpen blijft.

Een eenvoudige class

Laten we een klasse maken die "random walkers"  vertegenwoordigt.
We definiëren een class als volgt:

class Walker {

  constructor(x, y) {

    this.x = x;

    this.y = y;

  }

  bewegen() {

    this.x += random(-5, 5);

    this.y += random(-5, 5);

  }

  tonen() {

    fill(0);

    ellipse(this.x, this.y, 10, 10);

  }

}

Dit ziet er eigenlijk wel handig uit!

Merk op:

  •  Er zijn geen haakjes zijn na class Walker
  • Geen dubbele punten en komma’s.
  • De constructor() verzamelt alle eigenschappen
  • De constructor() verzamelt alle parameters.
  • De bewegen() en tonen() zijn het woordje function kwijt!

100 objecten uit één klasse?

Nu je weet hoe je één enkel object uit een klasse kunt maken, laten we dit concept uitbreiden door meerdere objecten te maken en deze in een array op te slaan.
Jawel, je kan ook objecten in een array stoppen!

let walkers = []; //array om alle objecten in op te slaan

let totalwalkers=100;

function setup() {

  createCanvas(400, 400);

  // De objecten aanmaken in setup()

  for (let i = 0; i < totalwalkers; i++) {

    walkers[i] = new Walker(random(width), random(height));

  }

}

function draw() {

  background(220);

  // De objecten doorlopen en de methoden voor elk object
 

  for (let i = 0; i < walkers.length; i++) {

    walkers[i].bewegen();

    walkers[i].tonen();

  }

}

Conclusie

Klassen vormen een essentieel onderdeel van geavanceerd programmeren en zijn een logische vervolgstap na het beheersen van enkele objecten. Ze dienen als een blauwdruk  voor het creëren van meerdere objecten met vergelijkbare eigenschappen en gedragingen. Dit maakt je code niet alleen gestroomlijnd, maar ook veelzijdiger, omdat je met klassen complexe systemen kunt modelleren, zoals een scherm vol met "random walkers".

Door het gebruik van klassen kun je je projecten naar een hoger niveau van complexiteit en interactiviteit tillen. Animatie, zie flowfield.
